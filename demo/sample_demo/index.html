How it works

          
The recording's chat turns (/run events) are re-played
             as a canvas animation and captured to a video file — no backend,
             no LLM, no regeneration.

          
Speed × — 1 is a human-comfortable
             reading pace. 2 plays twice as fast (the file is half
             as long); 0.5 is slower.

          
Format — detecting….
             MP4 is used when the browser supports it (Chrome / Edge / Safari);
             otherwise it falls back to WebM.

          
Recording happens in real time, so a longer conversation at ×1
             takes that long to capture. Keep this tab in the foreground while
             it runs.

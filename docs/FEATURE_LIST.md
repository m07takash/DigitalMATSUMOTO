# デジタルMATSUMOTO 機能一覧（概念レベル）

## 全体構成

```
┌─────────────────────────────────┐
│  UI／インターフェース層          │  人間やシステムとの接点
├─────────────────────────────────┤
│  アプリケーション層              │  実行制御・判断・分析
├─────────────────────────────────┤
│  コンテキストデザイン層          │  AIに与える文脈の設計
├─────────────────────────────────┤
│  データ／ナレッジ層              │  知識・記憶・履歴の管理
├─────────────────────────────────┤
│  インフラ／基盤層                │  LLM接続・認証・永続化
└─────────────────────────────────┘
```

---

## 1. インフラ／基盤層（つなぐ・守る・保つ）

| # | 機能 | 概要 |
|---|------|------|
| 1-1 | マルチLLM抽象化 | GPT/Gemini/Claude/Grokを同一インターフェースで呼び出す |
| 1-2 | マルチ画像生成抽象化 | DALL-E/Geminiの画像生成を同一インターフェースで呼び出す |
| 1-3 | トークン計量 | 各モデルのトークナイザーでプロンプト・応答のトークン数を計測する |
| 1-4 | テキストサニタイズ | LLM入力からJSONパース不能文字（NUL・孤立サロゲート・制御文字）を除去する |
| 1-5 | ベクトル埋め込み | テキストをEmbeddingモデルで数値ベクトルに変換する |
| 1-6 | 類似度計算 | コサイン距離等でベクトル間の意味的近さを算出する |
| 1-7 | ベクトルDB管理 | ChromaDBへのコレクション作成・検索・メタデータ更新 |
| 1-8 | セッション永続化 | チャット履歴・設定・ベクトルをファイルシステムに保存する |
| 1-9 | ファイルロック | 同一セッションへの並行書き込みを排他制御する |
| 1-10 | ステータス管理 | セッションのLOCK/UNLOCK/エラーをYAMLで管理しUIと同期する |
| 1-11 | ユーザー認証 | bcryptハッシュ・HMAC署名クッキーによるログイン認証 |
| 1-12 | アクセス制御 | ユーザーグループとAllowed設定による機能・エージェントの表示制御 |
| 1-13 | 環境設定 | setting.yaml / system.env による外部設定管理 |
| 1-14 | 画像MIMEタイプ判定 | ファイル拡張子から画像のMIMEタイプを動的に判定する |

## 2. データ／ナレッジ層（知識を蓄える・整理する）

| # | 機能 | 概要 |
|---|------|------|
| 2-1 | CSVデータ取り込み | CSVファイルからRAGチャンクデータを生成する |
| 2-2 | Notionデータ取り込み | NotionDBからプロパティ型に応じてRAGチャンクデータを生成する |
| 2-3 | 差分ベクトル化 | タイトル・キー・バリューに変更があるチャンクのみ再ベクトル化し、それ以外はメタデータのみ更新する |
| 2-4 | Privateフラグ | RAGチャンクに公開/非公開の属性を持たせる |
| 2-5 | Privateフラグマイグレーション | 既存データに一括でprivate=falseを付与する |
| 2-6 | カテゴリフィルタ | RAGデータをカテゴリ条件でフィルタリングしてコレクションに登録する |
| 2-7 | 会話履歴の構造化保存 | seq/sub_seq階層でプロンプト・応答・設定・ダイジェストを一括保存する |
| 2-8 | コンテンツファイル管理 | アップロード/生成ファイルをセッション配下に保存する |
| 2-9 | セッションアーカイブ | 古いセッションをZIP圧縮してアーカイブする |
| 2-10 | PostgreSQLエクスポート | セッション・対話・リファレンスをRDBにエクスポートする |
| 2-11 | フィードバック保存 | ユーザーのフィードバックをCSVまたはNotionに保存する |
| 2-12 | ユーザー対話保存 | ユーザーの発言傾向をCSVまたはNotionに保存する |
| 2-13 | マスターデータ管理 | ユーザー・RAG・プロンプトテンプレート・カテゴリ等のマスター定義 |

## 3. コンテキストデザイン層（AIに何を伝えるかを設計する）

### 人格コンテキスト

| # | 機能 | 概要 |
|---|------|------|
| 3-1 | システムプロンプト自動生成 | 人格設定（名前・性別・国籍・Big5・口調）からシステムプロンプトを動的に組み立てる |
| 3-2 | キャラクター定義 | 外部テキストファイルで経歴・価値観・一人称等の詳細な人格を記述する |
| 3-3 | Big5性格モデル | 5因子（開放性・誠実性・外向性・協調性・神経症傾向）を0〜1で設定しAIの振る舞いに反映する |
| 3-4 | 口調テンプレート | 「丁寧語」「武士語」「ギャル語」等の話し方スタイルをプロンプトに注入する |

### 知識コンテキスト（RAG）

| # | 機能 | 概要 |
|---|------|------|
| 3-5 | ヘッダーテンプレート | 知識のドメイン（Identity/Experience等）をAIに認識させるヘッダー文を定義する |
| 3-6 | チャンクテンプレート | 個々の知識チャンクにタイムスタンプ・類似度等の意味づけを加えて整形する |
| 3-7 | ログテンプレート | 知識チャンクの参照ログ形式を定義し分析・可視化に利用する |
| 3-8 | テキスト上限制御 | RAGコンテキストのドメインごとの文字数上限を設定する |
| 3-9 | タイムスタンプ付与 | チャンクに日付と「何日前か」の経過情報を付与し鮮度を伝える |
| 3-10 | 類似度ランキング | クエリとの類似度でチャンクを選択・順序付けする |
| 3-11 | メタデータ検索ボーナス | 日付条件に合致するチャンクに類似度ボーナスを加算する |
| 3-12 | コンテキストフィルタ | サービスID・ユーザーID・定義コード等でRAG検索範囲を絞り込む |
| 3-13 | Private Mode | private=trueのデータをRAG検索結果から除外する |

### 会話コンテキスト

| # | 機能 | 概要 |
|---|------|------|
| 3-14 | 会話メモリ選択 | 過去の会話履歴をトークン予算内でプロンプトに含める |
| 3-15 | メモリロールフィルタ | user/assistant/bothのいずれのロールの発言を含めるか選択する |
| 3-16 | メモリ優先度 | latest(最新優先)/oldest(古い順)/similar(類似度順)で会話メモリを選択する |
| 3-17 | メモリ類似度 | 現在のクエリと過去の会話の意味的近さで関連する記憶を選択する |
| 3-18 | 会話ダイジェスト | 長い会話履歴を要約して圧縮しコンテキストに含める |

### 状況コンテキスト

| # | 機能 | 概要 |
|---|------|------|
| 3-19 | シチュエーション設定 | 現在の日時・状況をプロンプトに注入する |
| 3-20 | 架空時間設定 | 通常の日時と異なる架空の時間設定を強い指示として注入する |

### 入力コンテキスト

| # | 機能 | 概要 |
|---|------|------|
| 3-21 | コンテンツコンテキスト生成 | アップロードファイル（テキスト/PDF/JSON/画像/音声）を解析しプロンプトに含める |
| 3-22 | Web検索コンテキスト | Web検索結果を「参考情報」としてプロンプトに付加する |

### プロンプト設計

| # | 機能 | 概要 |
|---|------|------|
| 3-23 | プロンプトテンプレート | 用途別（通常会話/考察/川柳/画像生成等）の指示テンプレートを管理する |
| 3-24 | プロンプト組み立て | 知識コンテキスト＋テンプレート＋ユーザー入力＋状況を連結して最終プロンプトを構築する |

## 4. アプリケーション層（判断する・実行する・分析する）

### 実行制御

| # | 機能 | 概要 |
|---|------|------|
| 4-1 | プラクティスチェイン | 複数ステップ（LLM→LLM、LLM→画像生成等）を順番に実行するパイプライン |
| 4-2 | ステップ間データ連携 | 前ステップの出力(OUTPUT)・コンテンツ(EXPORT)を後ステップの入力に渡す |
| 4-3 | エージェント切替 | チェインのステップごとに異なるエージェント（人格・モデル）を使い分ける |
| 4-4 | 設定上書き | 実行時にエージェントのエンジン設定等を動的に上書きする |
| 4-5 | ストリーミング応答 | LLMの応答をトークン単位でリアルタイムに返す |
| 4-6 | バックグラウンド実行 | UIをブロックせずバックグラウンドスレッドで実行し完了をポーリングする |
| 4-7 | セッションロック制御 | 実行中の二重実行を防止し、マルチチェインの中間ダイジェストによる誤UNLOCKを防ぐ |

### 自律判断

| # | 機能 | 概要 |
|---|------|------|
| 4-8 | Magic Word判定 | ユーザー入力のキーワード一致でHabit（振る舞い）を切り替える |
| 4-9 | Thinking Mode | AIが質問を分析してHabit・Web検索・RAGクエリ戦略・Book追加を動的に判定する |

### サポートエージェント

| # | 機能 | 概要 |
|---|------|------|
| 4-10 | RAGクエリ生成 | ユーザーの質問の深層心理を推定し、RAG検索に適した補助クエリを生成する |
| 4-11 | 日付抽出 | ユーザーの質問から日付情報を抽出しメタデータ検索条件に変換する |
| 4-12 | 会話ダイジェスト生成 | 会話履歴を要約してメモリ容量を圧縮する |
| 4-13 | セッション名生成 | 会話内容から15文字以内のスレッド名を自動生成する |
| 4-14 | ユーザー対話分析 | ユーザーの発言傾向を抽出・記録する |
| 4-15 | 画像批評 | 画像の概要・表現・印象をキュレーター視点で解説する |
| 4-16 | テキスト比較 | 2つの応答テキストを比較・評価する |

### 分析

| # | 機能 | 概要 |
|---|------|------|
| 4-17 | 知識活用分析 | 各RAGチャンクの質問類似度と回答類似度の差分から知識の貢献度を算出する |
| 4-18 | 次元削減可視化 | PCA/t-SNEでRAGチャンクの分布を2Dプロットする |
| 4-19 | エージェント比較分析 | 同じ質問に対する異なるエージェントの応答を比較・評価する |
| 4-20 | サポートエージェント評価 | テスト質問セットでRAGクエリ生成・日付抽出の速度・品質をベンチマークする |
| 4-21 | エシカルチェック | 応答を10項目の不適切表現カテゴリで評価する |

## 5. UI／インターフェース層（人間・外部システムとの接点）

### WebUI（Streamlit）

| # | 機能 | 概要 |
|---|------|------|
| 5-1 | チャット画面 | ユーザー/AI/詳細情報のチャット形式表示 |
| 5-2 | エンジン選択 | LLM/IMAGEGENのエンジンをドロップダウンで切り替える |
| 5-3 | Book選択 | 追加の知識ソースをマルチセレクトで選択する |
| 5-4 | Web検索トグル | Web検索の有効化とエンジン選択 |
| 5-5 | Private Mode/Thinking Modeトグル | チャット入力付近でモードを切り替える |
| 5-6 | Exec Setting | ストリーミング/メモリ/ダイジェスト等の実行オプション制御 |
| 5-7 | ファイルアップロード | テキスト/PDF/CSV/JSON/画像/音声の添付 |
| 5-8 | Detail Information | 実行情報・ログ・Thinking結果・RAGコンテキスト・Web検索結果の詳細表示 |
| 5-9 | フィードバック入力 | 各応答に対するカテゴリ・メモ・コメントの入力 |
| 5-10 | 分析ダッシュボード | 知識活用分析・エージェント比較の実行と結果表示 |
| 5-11 | セッション管理 | セッションの一覧・検索・切替・削除・アーカイブ |
| 5-12 | Markdown/PDFダウンロード | 会話履歴をMarkdownまたはPDF形式でエクスポートする |
| 5-13 | RAG管理 | RAGデータの更新・一覧表示 |
| 5-14 | ポーリング監視 | バックグラウンド実行中のステータスメッセージ・部分応答を2秒ごとに表示する |

### REST API（FastAPI）

| # | 機能 | 概要 |
|---|------|------|
| 5-15 | メッセージ送信API | /run でエージェント実行（LINE/Slack等外部サービスからの呼び出し対応） |
| 5-16 | セッション取得API | セッション一覧・履歴の取得 |
| 5-17 | エージェント情報API | エージェント一覧・エンジン一覧・フィードバック設定の取得 |
| 5-18 | フィードバック送信API | 外部からフィードバックを送信 |

# 設定ファイルとデプロイ構成

## 概要

システムの動作はYAML/ENV/JSONの3種類の設定ファイルで制御される。DockerベースのデプロイでStreamlit UIとFastAPIを同時起動できる。

## 仕組み

### 設定ファイル

| ファイル | 形式 | 役割 |
|---------|------|------|
| `setting.yaml` | YAML | フォルダパス、Web検索モデル等の構造設定 |
| `system.env` | ENV | APIキー、認証設定、マスターファイル名 |
| `user/common/agent/*.json` | JSON | エージェント定義（人格・LLM・ナレッジ） |
| `user/common/mst/*.json` | JSON | マスターデータ（ユーザー・RAG・プロンプト） |

### 主要環境変数

| 変数名 | 説明 |
|--------|------|
| `OPENAI_API_KEY` | OpenAI APIキー（必須: Embedding使用） |
| `GEMINI_API_KEY` / `ANTHROPIC_API_KEY` / `XAI_API_KEY` | 各LLMのAPIキー |
| `LOGIN_ENABLE_FLG` | ログイン認証有効化（Y/N） |
| `EMBEDDING_MODEL` | 埋め込みモデル（デフォルト: text-embedding-3-large） |
| `USER_MST_FILE` / `RAG_MST_FILE` / `PROMPT_TEMPLATE_MST_FILE` | マスターファイル名 |

### ディレクトリ構造

```
DigitalMATSUMOTO/
├── user/common/agent/       # エージェント設定JSON
├── user/common/practice/    # プラクティス設定JSON
├── user/common/mst/         # マスターデータ
├── user/common/rag/chromadb/  # ベクトルDB
├── user/common/rag/pages/   # RAGページデータ
├── user/common/csv/         # RAG用CSVデータ
├── user/session*/           # セッション（チャット履歴）
└── user/archive/            # アーカイブZIP
```

### Docker構成

`Dockerfile`はUbuntu 22.04ベースで、Python3、MeCab（NEologd辞書）、日本語フォントを含む。ポート8501（Streamlit）、8899（FastAPI）を公開する。

## 設定場所

- `setting.yaml`: プロジェクトルート
- `system.env`: プロジェクトルート（`system.env_sample`からコピーして作成）
- `Dockerfile`: プロジェクトルート

## 設定例

```bash
# コンテナ起動
docker run -d --name digimatsumoto \
  -p 8501:8501 -p 8899:8899 \
  -v $(pwd):/app/DigitalMATSUMOTO \
  digimatsumoto
```

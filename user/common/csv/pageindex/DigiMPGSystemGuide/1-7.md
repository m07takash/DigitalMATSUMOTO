# 認証・アクセス制御

## 概要

`WebDigiMatsuAgent.py`でStreamlit UI向けのログイン認証、Cookie永続化、グループベースのアクセス制御を実装している。認証ソースは **JSON / RDB(PostgreSQL)** を環境変数で切替可能。bcryptによるパスワードハッシュ化とHMACトークン検証を使用する。

## 仕組み

### ログイン認証

`LOGIN_ENABLE_FLG=Y`で有効化される。`ensure_login()`関数がエントリポイント。

1. `st.session_state`にログイン情報があればスキップ
2. Cookie(`digim_auth`)からHMACトークンを検証し、有効ならセッション復元
3. どちらもなければログインフォームを表示し、`st.stop()`で後続処理をブロック

### 認証ソースの切替（`DigiM_Auth.py`）

`LOGIN_AUTH_METHOD`環境変数で切替（既定: `JSON`）。

| 値 | ストレージ |
|----|-----------|
| `JSON` | `USER_MST_FILE` で指定したJSONファイル（`user/common/mst/users.json`等） |
| `RDB` | PostgreSQL の `digim_users` テーブル（初回アクセス時に `CREATE TABLE IF NOT EXISTS` で自動作成） |

`DigiM_Auth` モジュールが `load_user_master()` / `save_user_master()` を統一インターフェースで提供。`psycopg2` は遅延importなのでJSONモードでは PostgreSQL関連エラーは発生しない。

`digim_users` テーブルのスキーマ:
| カラム | 型 | 説明 |
|-------|----|------|
| `user_id` | `TEXT PRIMARY KEY` | ユーザーID |
| `name` | `TEXT` | 表示名 |
| `pw` | `TEXT` | bcryptハッシュ（または平文を初回ログインで自動移行） |
| `group_cd` | `JSONB` | 所属グループの配列 |
| `agent` | `TEXT` | デフォルトエージェントファイル名 |
| `allowed` | `JSONB` | UI機能の表示/非表示マップ |

### パスワード管理

- 初回ログイン時に平文パスワードを検出すると、自動的にbcryptハッシュ(`$2b$`等)に変換して保存
- `DigiM_Util.verify_password()`が平文・ハッシュの両方に対応
- ログイン画面のChange Passwordタブからパスワード変更可能（JSON / RDB両対応）

### Cookie認証

- `_make_auth_token(user_id)`: HMAC-SHA256でuser_idに署名し`user_id:signature`形式のトークンを生成
- `_verify_auth_token(token)`: 署名を検証しuser_idを返す。`hmac.compare_digest()`でタイミング攻撃を防止
- Cookie有効期限: 7日（`_COOKIE_EXPIRY_DAYS`）
- シークレットキー: 環境変数`COOKIE_SECRET`
- ログアウト時はCookieを期限切れで上書きし、次回再描画ではCookie復元をスキップする（`_just_logged_out` フラグ）

### グループ・権限管理

ユーザーマスターの`Group`フィールドは **配列** でアクセスレベルを制御する（後方互換のため文字列も受け付け、内部でリスト正規化）。

| 設定 | 権限 |
|------|------|
| `["Admin"]`（配列内に`Admin`があれば） | 全ユーザーの履歴閲覧、全エージェント選択可 |
| 個別グループ群（例: `["Sales", "Marketing"]`） | エージェントの`GROUP`と **OR一致** したエージェントが選択可 |

`get_display_agents(user_groups)` は文字列・リストの両方を受け取り、リストの場合はOR条件でエージェントをフィルタする。

`Allowed`フィールドで各UI機能の表示/非表示を個別制御する（RAG Management, Exec Setting, Feedback, Analytics等、計13項目）。

## 設定場所

- 認証有効化: `system.env`の`LOGIN_ENABLE_FLG`
- 認証ソース: `system.env`の`LOGIN_AUTH_METHOD`（`JSON` / `RDB`）
- ユーザーマスター（JSON）: `user/common/mst/`配下の`users.json`（`USER_MST_FILE`で指定）
- ユーザーマスター（RDB）: PostgreSQLの`digim_users`テーブル
- Cookie秘密鍵: `system.env`の`COOKIE_SECRET`

## 設定例

JSON形式:
```json
{
  "USER0001": {
    "Name": "ユーザー名",
    "PW": "パスワード",
    "Group": ["User"],
    "Agent": "agent_10Sample.json",
    "Allowed": {
      "RAG Management": true,
      "Exec Setting": true,
      "WEB Search": true,
      "Book": true
    }
  },
  "USER0002": {
    "Name": "営業＋マーケ兼務",
    "PW": "...",
    "Group": ["Sales", "Marketing"],
    "Agent": "DEFAULT"
  }
}
```

RDB初期登録（PostgreSQL）:
```sql
INSERT INTO digim_users (user_id, name, pw, group_cd, agent, allowed) VALUES
  ('USER0001', 'ユーザー名', 'password', '["User"]'::jsonb, 'agent_10Sample.json',
   '{"RAG Management": true, "WEB Search": true}'::jsonb)
ON CONFLICT (user_id) DO UPDATE
SET name=EXCLUDED.name, pw=EXCLUDED.pw, group_cd=EXCLUDED.group_cd,
    agent=EXCLUDED.agent, allowed=EXCLUDED.allowed;
```

# ベクトル埋め込みと類似度計算

## 概要

テキストをベクトル空間に変換し、ベクトル間の距離を計算する機能群。RAG検索、会話メモリの類似度評価、ナレッジ参照の関連度算出に使用される。

## 仕組み

### 埋め込み関数

**`embed_text(text)`**: 単一テキストをOpenAI Embedding APIでベクトル化する。8192トークンを超えるテキストは自動的に切り詰める。tiktokenで事前にトークン数を計算する。

**`embed_texts_batch(texts)`**: 複数テキストを1回のAPI呼び出しでバッチベクトル化する。レスポンスはindex順でソートして返す。`DigiM_Execute.py`のクエリベクトル化やRAGチャンク登録で使用される。

### クライアント管理

`_embed_client`と`_embed_enc`がシングルトンで管理され、初回呼び出し時にのみ`OpenAI`クライアントとtiktokenエンコーダを生成する。使用モデルは環境変数`EMBEDDING_MODEL`で指定する。

### 類似度計算

**`calculate_similarity_vec(vec1, vec2, logic)`**: 距離計算のメインディスパッチャ。

| logic | 計算方式 |
|-------|---------|
| `Cosine` | コサイン距離（1 - コサイン類似度） |
| `Euclidean` | ユークリッド距離（ミンコフスキー p=2） |
| `Manhattan` | マンハッタン距離（ミンコフスキー p=1） |
| `Chebyshev` | チェビシェフ距離（ミンコフスキー p=inf） |

**`calculate_cosine_distance(vec1, vec2)`**: コサイン類似度をドット積と大きさから手計算し、1から引いて距離に変換する。ゼロベクトルの場合は0を返す。

### ベクトル永続化

`save_vectext_to_npy()` / `read_vectext_to_npy()`でNumPy `.npy`形式でファイル保存・読込する。メモリマップド読込(`mmap`)にも対応している。

## 設定場所

- 埋め込みモデル: `system.env`の`EMBEDDING_MODEL`
- 類似度ロジック: エージェントJSON `ENGINE.*.MEMORY.similarity_logic`

## 設定例

```env
EMBEDDING_MODEL=text-embedding-3-large
```

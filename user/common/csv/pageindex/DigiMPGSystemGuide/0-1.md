# システムアーキテクチャ全体像

## 概要

Digital MATSUMOTOは、人間の知識をAIが再現することを目的としたデジタルツインシステムである。特定のLLMに依存せず、LangChain等のライブラリを使わずにLLM APIを直接呼び出す設計思想を持つ。

## 仕組み

5層アーキテクチャで構成される。

```
[Streamlit UI / FastAPI / Jupyter Notebook]   ← プレゼンテーション層
          |
   DigiM_Execute.py        ← オーケストレーション層（Thinking Mode判定含む）
          |
   DigiM_Agent.py / DigiM_Context.py / DigiM_Session.py  ← ドメイン層
          |                        |
   DigiM_FoundationModel.py    [ChromaDB / PageIndex]    ← 基盤モデル層 / データ層
          |
[GPT / Gemini / Claude / Grok / Llama / Azure OpenAI]    ← 外部LLM層
```

**主要モジュール一覧:**

| モジュール | 役割 |
|-----------|------|
| `DigiM_Execute.py` | メイン実行エンジン（パイプライン制御） |
| `DigiM_Agent.py` | エージェント設定（テンプレート + persona上書き） |
| `DigiM_AgentPersona.py` | エージェントペルソナのマスタ（Excel/RDB） |
| `DigiM_Context.py` | コンテキスト生成・RAG処理 |
| `DigiM_Session.py` | セッション・チャット履歴管理 |
| `DigiM_FoundationModel.py` | LLM抽象化（GPT/Gemini/Claude/Grok/Llama/Azure OpenAI） |
| `DigiM_Tool.py` | ツール群（Web検索・分析・ペルソナ統合等） |
| `DigiM_Util.py` | 共通ユーティリティ（埋め込み・トークン・サニタイズ等） |
| `DigiM_Auth.py` | ログイン認証・ユーザーマスタ（JSON/RDB切替） |
| `DigiM_Notion.py` | Notion API連携 |
| `DigiM_DB_Export.py` | PostgreSQLエクスポート・ベクトル化 |
| `DigiM_VAnalytics.py` | 知識活用分析 |
| `DigiM_GeneFeedback.py` | ユーザーフィードバック（CSV/Notion保存） |
| `DigiM_UserMemory.py` | ユーザーメモリ（History/Nowaday/Persona）ストレージ抽象化 |
| `DigiM_UserMemorySetting.py` | ユーザーメモリのOn/Off2階層解決 |
| `DigiM_UserMemoryBuilder.py` | ユーザーメモリのコンテキスト合成（タグ×時間ハイブリッド検索） |
| `DigiM_GeneUserMemory.py` | ユーザーメモリ生成・差分マージ・バックフィル（CLI付き） |
| `DigiM_UserMemoryExplorer.py` | User Memory Explorerバックエンド |
| `DigiM_Scheduler.py` | APSchedulerベースのバックグラウンドスケジューラ |
| `DigiM_ScheduledJobs.py` | スケジュールジョブのマスタCRUD |
| `DigiM_JobRegistry.py` | バックグラウンドスレッドの登録・キャンセル |
| `DigiM_UrlFetch.py` | チャット入力中のhttp(s)リンク自動取得（サブページクロール対応） |
| `WebDigiMatsuAgent.py` | Streamlit WebUI |
| `DigiM_API.py` | FastAPIエンドポイント |

**データフロー:** ユーザー入力 → URL自動取得 → Thinking Mode（任意） → コンテキスト生成（RAG[Vector/PageIndex] + ユーザーメモリ + Web検索） → プロンプト構築 → テキストサニタイズ → LLM実行 → レスポンス保存・類似度評価 → ダイジェスト生成（バックグラウンド）

**主要な機能:**
- **Thinking Mode**: AIがクエリを分析し、Habit / Web検索 / RAGクエリ生成 / Book追加を動的に判定
- **PageIndex RAG**: ベクトル検索に代わり、LLMがページインデックスから関連ページを選択
- **ユーザーメモリ**: Persona/Nowaday/Historyの3層でユーザー理解を蓄積し、Knowledge直前にコンテキスト注入

## 設定場所

- システム設定: `setting.yaml`
- 環境変数: `system.env`（`system.env_sample` をコピー）
- エージェント定義: `user/common/agent/*.json`
- プロンプトテンプレート: `user/common/mst/prompt_templates.json`

## 設定例

```yaml
# setting.yaml（フォルダパス定義）
USER_FOLDER: "user/"
AGENT_FOLDER: "user/common/agent/"
RAG_FOLDER_DB: "user/common/rag/chromadb/"
USER_MEMORY_FOLDER: "user/common/user_memory/"
```

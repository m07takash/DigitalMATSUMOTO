# 4-7. スケジューラ（cron駆動の汎用ジョブ）

## 概要
RAG更新・ユーザーメモリ更新・エージェント実行などのバックグラウンドジョブを cron 書式で定期実行する汎用スケジューラ。ジョブ定義は `user/common/mst/scheduled_jobs.json` に永続化され、WebUI上部の **Scheduler** メニュー（Chat / Knowledge Explorer / User Memory Explorer の隣）から編集できる。再起動なしで反映可能（**Reload Schedulers**）。バックエンドは `DigiM_Scheduler.py` + `DigiM_ScheduledJobs.py`。

## 仕組み

### ジョブ種別（`kind`）

| kind | 内容 | セッション |
|------|------|------|
| `rag_update` | `DigiM_Context.generate_rag()` を呼びRAGを再ベクトル化。`USER_MEMORY_HISTORY_AUTO_SAVE_FLG=Y` の場合は未保存セッションのHistoryも併せて自動保存。 | 作成しない |
| `user_memory_nowaday` | 全ユーザーに対し当月Nowadayを生成 → Personaに差分マージ（`DigiM_GeneUserMemory.build_nowaday_for_all_users` → `merge_persona`）。 | 作成しない |
| `agent_run` | `params.agent_file` のエージェントを `params.user_input` で実行（`DigiMatsuExecute_Practice`）。`owner_user_id` を USER_ID、service_id=`Scheduler`、session_id=`SCH<日時>`、名前=`[Scheduler] <ジョブ名>` で **新規セッションを発番** し応答をチャット履歴に保存。`params.engine` で LLM 上書きも可能。 | `SCH<日時>` を新規発番 |

### cron書式
- `"off"` または `enabled=false` → 登録されない
- プリセット: `"daily"`（毎日03:00）/ `"weekly"`（毎週月曜03:00）/ `"monthly"`（毎月1日03:00）
- 5フィールドのcron文字列（例: `"0 3 1 * *"`）

タイムゾーンは `system.env` の `TIMEZONE`（既定 `Asia/Tokyo`）。APScheduler 未インストール環境では cron 起動はスキップされるが、**Run Now** による手動実行は引き続き可能。

### WebUI 操作（Scheduler 画面）
- 一覧: ジョブごとに `name` / `kind` / `cron` / `enabled` / `owner_user_id` / `last_run` / `last_status` / `last_session_id`（agent_runのみ）/ `last_error` を表示
- 各行: **Edit** / **Run Now**（即時1回実行）/ **Enable/Disable** / **Delete**
- 全体: **Add New Job** / **Reload Schedulers**（cron変更後の反映）
- 所有者ユーザー（`owner_user_id`）は保存時のログインユーザーが自動セット。`agent_run` のセッションはこのユーザーに紐付く。

### 実行結果の記録
`_run_job` が `update_run_result` で `last_run` / `last_status`（running/success/error）/ `last_error` を `scheduled_jobs.json` に書き戻す。`agent_run` のみ `last_session_id` も記録され、チャット履歴から応答を確認できる。

## 設定場所
- 実装: `DigiM_Scheduler.py`（APScheduler ラッパ）/ `DigiM_ScheduledJobs.py`（マスタJSON入出力）
- マスタJSON: `user/common/mst/scheduled_jobs.json`
- 環境変数: `system.env` の `TIMEZONE`（既定 `Asia/Tokyo`）、`USER_MEMORY_HISTORY_AUTO_SAVE_FLG`（`rag_update` 連動のHistory自動保存）
- 権限: `users.json` の `Allowed["Scheduler"]`（Adminは常時可）

## 設定例
```json
[
  {"job_id": "rag_daily", "name": "RAG Daily Update", "kind": "rag_update",
   "cron": "daily", "enabled": true, "owner_user_id": "Admin"},
  {"job_id": "umnowaday", "name": "Nowaday Monthly", "kind": "user_memory_nowaday",
   "cron": "0 3 1 * *", "enabled": true, "owner_user_id": "Admin"},
  {"job_id": "morning_brief", "name": "Morning Brief", "kind": "agent_run",
   "cron": "0 8 * * *", "enabled": true, "owner_user_id": "RealMatsumoto",
   "params": {"agent_file": "agent_10Sample.json", "user_input": "今日のトピックを3つ挙げて", "engine": ""}}
]
```

# 画像生成・マルチモーダル対応

## 概要

DALL-EおよびGeminiによる画像生成と、各LLMへの画像入力（マルチモーダル）をサポートする。生成画像はTEMPフォルダに保存後、セッションのcontentsフォルダにコピーされる。

## 仕組み

### 画像生成

**DALL-E (`generate_image_dalle`):**
- メモリ + ユーザープロンプトをJSON文字列化し、先頭3000文字を送信（送信前に`_sanitize_messages()`で不正文字を除去）
- `output_format`デフォルトはPNG（`"output_format": "png"`）。`b64_json`形式でレスポンスを受け取りファイル保存
- `model["PARAMETER"]`でサイズ等を制御

**Gemini (`generate_image_gemini`):**
- `response_modalities=["TEXT", "IMAGE"]`で画像生成を指示
- `aspect_ratio`パラメータ対応（ImageConfig経由）
- レスポンスの各partからテキストと画像を分離して保存

### マルチモーダル入力

各LLM関数は`image_paths`引数で画像ファイルを受け取り、base64エンコードしてプロンプトに含める。

| LLM | 画像形式 |
|-----|---------|
| GPT | `image_url`型（data URI） |
| Gemini | `inlineData`型（mimeType + data） |
| Claude | `image`型（base64 source） |
| Grok | xai_sdk.chat.image関数 |

### MIMEタイプ自動判定

`_get_image_mime(image_path)`（`DigiM_FoundationModel.py`）がPythonの`mimetypes.guess_type()`を使用してファイル拡張子からMIMEタイプを自動判定する。判定できない場合やイメージ以外のMIMEタイプが返された場合は`image/png`をデフォルトとする。全LLM関数（GPT/Gemini/Claude/Grok等）で画像をbase64エンコードする際に使用され、各LLMのAPI仕様に合わせた正しいMIMEタイプ（`image/jpeg`, `image/png`, `image/webp`等）が自動的に設定される。

## 設定場所

- エージェントJSON: `ENGINE.IMAGEGEN`セクション
- TEMPフォルダ: `setting.yaml`の`TEMP_FOLDER`

## 設定例

```json
{
  "IMAGEGEN": {
    "DEFAULT": "DALL-E",
    "DALL-E": {
      "FUNC_NAME": "generate_image_dalle",
      "MODEL": "gpt-image-1",
      "PARAMETER": {"quality": "high", "output_format": "png"},
      "TOKENIZER": "tiktoken",
      "MEMORY": {"limit": 5000, "role": "both", "priority": "latest", "similarity_logic": "cosine", "digest": "Y"}
    }
  }
}
```

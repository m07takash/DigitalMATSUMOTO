# 画像生成・マルチモーダル対応

## 概要

OpenAI DALL-E / Gemini / Azure OpenAI Service による画像生成と、各LLMへの画像入力（マルチモーダル）をサポートする。生成画像はTEMPフォルダに保存後、セッションの `contents/` フォルダにコピーされる。

## 仕組み

### 画像生成

| 関数名 | 説明 |
|--------|------|
| `generate_image_dalle` | OpenAI DALL-E / gpt-image-1（`output_format` 既定 `png`、`b64_json` でファイル保存） |
| `generate_image_gemini` | Gemini（`response_modalities=["TEXT","IMAGE"]`、`aspect_ratio` 対応） |
| `generate_image_azure_dalle` | Azure OpenAI 上の dall-e / gpt-image-* デプロイ（`MODEL` にdeployment名） |

DALL-E / Azure DALL-E はメモリ + ユーザープロンプトをJSON文字列化し、`_sanitize_messages()` で不正文字を除去後、先頭3000文字を送信する。Geminiはレスポンスの各partからテキストと画像を分離して保存する。

### マルチモーダル入力

各LLM関数は `image_paths` 引数で画像ファイルを受け取り、`dmu.encode_image_file()` でbase64エンコードしてプロンプトに含める。

| LLM | 画像形式 |
|-----|---------|
| GPT / Azure OpenAI | `image_url` 型（data URI） |
| Gemini | `inlineData` 型（mimeType + data） |
| Claude | `image` 型（base64 source） |
| Grok | `xai_sdk.chat.image` |

### MIMEタイプ自動判定

`_get_image_mime(image_path)` がPythonの `mimetypes.guess_type()` を使用してファイル拡張子からMIMEタイプを自動判定する。判定不能やイメージ以外なら `image/png` をデフォルトとする。全LLM関数で画像base64化時に使用され、`image/jpeg` / `image/png` / `image/webp` 等を自動設定する。

## 設定場所

- エージェントJSON: `ENGINE.IMAGEGEN` セクション
- TEMPフォルダ: `setting.yaml` の `TEMP_FOLDER`
- Azure共通設定: `system.env` の `AZURE_OPENAI_*`

## 設定例

```json
{
  "IMAGEGEN": {
    "DEFAULT": "GPT-Image",
    "GPT-Image": {
      "NAME": "GPT-Image-1",
      "FUNC_NAME": "generate_image_dalle",
      "MODEL": "gpt-image-1",
      "PARAMETER": {"size": "1024x1024", "quality": "high"}
    },
    "nano-banana-2": {
      "NAME": "nano-banana-2",
      "FUNC_NAME": "generate_image_gemini",
      "MODEL": "gemini-3.1-flash-image-preview",
      "PARAMETER": {"aspect_ratio": "1:1"}
    },
    "GPT-Image-Azure": {
      "NAME": "GPT-Image-Azure",
      "FUNC_NAME": "generate_image_azure_dalle",
      "MODEL": "gpt-image-1",
      "PARAMETER": {"size": "1024x1024", "quality": "high"}
    }
  }
}
```

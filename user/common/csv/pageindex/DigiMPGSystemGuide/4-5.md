# 4-5. 画像批評とテキスト比較

## 概要
画像に対するAI批評と、2つのテキストの比較分析を行うサポートエージェント機能である。画像生成チェインと組み合わせることで、生成画像の品質評価ワークフローを構築できる。

## 仕組み

### ArtCritic（画像批評）
`DigiM_Tool.py`の`art_critics`関数が処理を担う。

1. **エージェント設定**: デフォルトで`agent_52ArtCritic.json`を使用。プラクティスJSONからプロンプトテンプレートコード（`Art Critic`等）を取得する。
2. **画像入力**: `image_paths`リストで画像ファイルパスを受け取り、LLMのマルチモーダル入力として渡す。各LLMプロバイダ（GPT/Gemini/Claude等）がbase64エンコードされた画像を解析する。
3. **批評生成**: プロンプトテンプレートの指示に基づき、画像の内容・構図・品質等を批評するテキストを生成する。
4. **コンテンツコンテキストでの利用**: `DigiM_Context.py`の`get_text_content`では、画像ファイルがアップロードされた際に`SUPPORT_AGENT.ART_CRITICS`で指定されたエージェントが自動的に画像を解釈し、テキストコンテキストに変換する。

### チェイン利用例（IMAGEGEN -> ArtCritic）
プラクティスの2ステップチェインで利用される典型的なパターン:
1. ステップ1（IMAGEGEN）: ユーザーのプロンプトから画像を生成
2. ステップ2（LLM）: 生成画像を`EXPORT_1`で参照し、ArtCriticエージェントが批評

### compare_texts（テキスト比較）
`DigiM_Tool.py`の`compare_texts`関数が処理を担う。

1. **エージェント設定**: `agent_53CompareTexts.json`を使用
2. **入力**: 2つのテキスト（`head1/text1`、`head2/text2`）と比較観点（`query_compare`）を受け取る
3. **プロンプト構成**: `{prompt_template}\n\n[{head1}]\n{text1}\n\n[{head2}]\n{text2}`の形式で比較プロンプトを構築
4. **比較分析**: LLMが2つのテキストの差異や類似点を分析し、結果を返す
5. **カスタム比較**: `query_compare`を指定すると、デフォルトテンプレートの代わりにカスタムの比較観点を使用できる

## 設定場所
- ArtCritic: エージェントJSONの`SUPPORT_AGENT.ART_CRITICS`
- CompareTexts: `agent_53CompareTexts.json`
- プロンプトテンプレート: マスターJSONの`Art Critic`、`Compare Texts`

## 設定例
```json
{
  "SUPPORT_AGENT": {
    "ART_CRITICS": "agent_52ArtCritic.json"
  }
}
```

# エージェント人格とキャラクター定義

## 概要
エージェントJSONに定義されたPERSONALITYフィールドとCHARACTERテキストから、LLMのシステムプロンプトを自動生成する機能。エージェントの名前、役割、性格、口調を含む一貫したペルソナを構築する。

## 仕組み

### エージェントの初期化（`__init__` / `set_property`）
`DigiM_Agent`クラスのコンストラクタで`agent_file`（JSONファイル名）を読み込み、`set_property()`で各プロパティを設定する。ENGINEのネスト構造はDEFAULT指定に基づきフラット化される。

### Persona上書き（`DigiM_AgentPersona`）
`DigiM_Agent(agent_file, persona=...)` でpersona dictを渡すと、テンプレートに対して以下のフィールドが上書きされる（`DigiM_Agent._apply_persona()`）:
- 上書き: `NAME` / `ACT` / `PERSONALITY`（`character_text` または `character_file` を含む）/ `HABIT`（`["ALL"]` 以外なら名前ホワイトリストでフィルタ）/ `KNOWLEDGE`（`["ALL"]` 以外なら `RAG_NAME` でフィルタ）/ `DEFINE_CODE`
- 不変: `ENGINE` / `SUPPORT_AGENT` / `BOOK` / `SKILL` / `FEEDBACK` / `GROUP` / `ORG` / `PERSONA_FILES`

Persona dict は `DigiM_AgentPersona.find_personas_by_org(agent_org, ...)` で取得（ExcelまたはRDBバックエンド）。グローバルは環境変数 `AGENT_PERSONA_SOURCE` (`EXCEL`/`RDB`/`BOTH`)、エージェントJSONの `PERSONA_SOURCE` フィールドで **per-agent上書き** も可能。複数ペルソナの並列実行・自動選定を含む詳細は **3-9** を参照。

### システムプロンプト自動生成（`set_system_prompt`）
`set_system_prompt()`が以下の順序でプロンプトを組み立てる。

1. **名前**: `NAME`が設定されていれば「あなたの名前は「{NAME}」です。」
2. **役割**: `ACT`が設定されていれば「{ACT}として振る舞ってください。」
3. **PERSONALITYフィールド**:
   - `SEX`: 性別
   - `BIRTHDAY`: 誕生日
   - `IS_ALIVE`: 存命状態
   - `NATIONALITY`: 国籍
   - `BIG5`: ビッグファイブ性格特性（詳細は3-2参照）
   - `LANGUAGE`: 使用言語
   - `SPEAKING_STYLE`: 口調テンプレート（詳細は3-3参照）
4. **CHARACTER**: テキストファイル名（`.txt`/`.md`で終わる場合）またはインライン文字列。ファイルの場合は`CHARACTER_FOLDER`から読み込む。「あなたのキャラクター設定:」として注入される。

## 設定場所
- エージェント定義: `user/common/agent/agent_*.json`（サンプルは `agent_10Sample.json`）
- キャラクターファイル: `setting.yaml`の`CHARACTER_FOLDER`内の`.txt`/`.md`ファイル
- プロパティキー: `NAME`, `ACT`, `PERSONALITY`（`SEX`, `BIRTHDAY`, `IS_ALIVE`, `NATIONALITY`, `BIG5`, `LANGUAGE`, `SPEAKING_STYLE`, `CHARACTER`）

## 設定例
```json
{
  "NAME": "デジタルコンサル",
  "ACT": "コンサルティングファームのパートナー",
  "PERSONALITY": {
    "SEX": "女性",
    "BIRTHDAY": "01-Jan-1980",
    "IS_ALIVE": true,
    "NATIONALITY": "Japanese",
    "LANGUAGE": "日本語",
    "SPEAKING_STYLE": "Polite",
    "CHARACTER": "Sample.txt"
  }
}
```

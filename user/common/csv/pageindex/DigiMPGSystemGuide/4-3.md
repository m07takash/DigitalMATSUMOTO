# 4-3. 自律判断（Magic Word・Thinking Mode）

## 概要
ユーザーの質問内容に応じて実行モードを自動的に切り替える仕組みである。Magic Wordによるキーワードマッチングと、Thinking Modeによるai駆動の判断の2つのアプローチがある。

## 仕組み

### Magic Word判定
`DigiM_Agent.py`の`set_practice_by_command`メソッドが処理を担う。

1. エージェントの`HABIT`辞書内の各Habitキーを走査
2. 各Habitに定義された`MAGIC_WORDS`リストとユーザークエリを比較
3. クエリ内にマジックワードが含まれていれば、そのHabitキーを返す
4. マッチしなければ`DEFAULT`を返す

選択されたHabit内の`PRACTICE`で指定されたプラクティスJSONが実行される。

### Thinking Mode
`DigiM_Execute.py`の`_run_thinking_agent()`関数とThinking Agent（`DigiM_Tool.py`の`thinking_agent`）で実装される。

#### `_run_thinking_agent()`の処理フロー
1. **入力情報の整形**: Habit一覧（キーとマジックワード）、Book一覧（RAG名）、会話ダイジェスト、シチュエーション情報をコンテキストとして準備
2. **Thinking Agent実行**: サポートエージェント設定の`THINKING`で指定されたエージェント（標準サンプル: `agent_59Thinking.json`、Digital MATSUMOTO本体: `agent_70DigiMThinking.json`）がクエリを分析し、JSONで実行パラメータを返す
3. **結果の適用**: 返却JSONから以下を抽出・設定
   - `habit`: 使用するHabitキー（Magic Wordより優先）
   - `web_search` / `web_search_engine`: Web検索の要否とエンジン
   - `rag_query_gene`: RAGクエリ生成の要否
   - `books`: 自動追加するBookのRAG名リスト
   - `reasoning`: 判断理由テキスト
4. **thinking_logの構築**: エージェントファイル名、モデル名、実行時間、入力/出力トークン数、reasoning、判定結果をログとして記録。このログは`execution["_THINKING_LOG"]`経由でチェイン実行に渡され、Detail Informationの【Thinking】セクションに表示される

#### Thinking Targets（WebUIでの判定範囲制御）
WebUIサイドバーの **Thinking Targets** multiselect で、Thinking Agentに任せる判定対象を選べる。

| 選択肢 | 効果 |
|--------|------|
| `Habit` | Habit切替を委譲 |
| `Web Search` | Web検索の要否・エンジンを委譲 |
| `RAG Query` | RAGクエリ生成の要否を委譲 |
| `Books` | 自動追加する Book を委譲 |
| `Personas` | **ORG が定義されたエージェントのみ表示**。`chain.PERSONAS="THINKING"` ステップで `agent_54PersonaSelector.json` による自動選定を有効化（Phase 7）。Thinking Mode ON かつ `Personas` 選択時のみ **Max Personas** 入力欄が出現（既定は `setting.yaml` の `MAX_PERSONAS=3`） |

`Personas` を外すと PersonaSelector はスキップされ `chain.PERSONAS="THINKING"` ステップは `in_personas` にフォールバックする。エージェント切替で `ORG` がなくなった場合は `Personas` が選択肢から外れ、保存済み選択も自動的にドロップされる。

#### フォールバック動作
Thinking Modeの結果が空（habitが含まれない）の場合、Magic Word判定にフォールバックする。これにより、Thinking Agentが判断できなかった場合でも従来のキーワードマッチングで適切なHabitが選択される。

### 優先順位
`DigiMatsuExecute_Practice`内での判定順序:
1. Thinking Modeが有効かつ結果にhabitが含まれる場合 -> Thinking結果を採用
2. それ以外でMagic Word判定が有効な場合 -> Magic Word結果を採用
3. いずれもマッチしなければ -> DEFAULTを使用

## 設定場所
- エージェントJSON: `HABIT`辞書、`SUPPORT_AGENT.THINKING`
- 実行設定: `execution["MAGIC_WORD_USE"]`、`execution["THINKING_MODE"]`

## 設定例
```json
{
  "HABIT": {
    "DEFAULT": {"PRACTICE": "practice_01Default.json", "MAGIC_WORDS": []},
    "IMAGE": {"PRACTICE": "practice_02Image.json", "MAGIC_WORDS": ["画像生成", "イラスト"]}
  },
  "SUPPORT_AGENT": {
    "THINKING": "agent_70DigiMThinking.json"
  }
}
```

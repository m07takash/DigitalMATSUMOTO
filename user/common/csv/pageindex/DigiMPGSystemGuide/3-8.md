# 3-8. プロンプトテンプレートと組み立て

## 概要
プロンプトテンプレートは、LLMへの指示内容を定型化する仕組みである。マスターファイルからテンプレートコードで取得し、ナレッジ・クエリ・シチュエーションと結合して最終プロンプトを組み立てる。

## 仕組み

### テンプレート取得
`DigiM_Context.py`の`set_prompt_template`関数が処理を担う。

1. `prompt_template_mst_file`（環境変数`PROMPT_TEMPLATE_MST_FILE`で指定）からマスターJSONを読み込む
2. `PROMPT_TEMPLATE`セクションから`prompt_temp_cd`キーで該当テンプレートを取得
3. テンプレートが存在すれば末尾に改行を付加して返す

`DigiM_Agent.py`の`set_prompt_template`メソッドがこの関数をラップしており、エージェントから呼び出される。

### 最終プロンプトの組み立て
`DigiM_Execute.py`のLLM実行ステップで最終プロンプトが構築される（ユーザーメモリコンテキストはこの直前にプロンプト先頭へ別途挿入される。詳細は3-10）。

**LLMの場合:**
```
query = knowledge_context + prompt_template + user_query + situation_prompt
```

**IMAGEGEN等の場合:**
```
query = prompt_template + user_query + situation_prompt
```

各要素の役割:
- `knowledge_context`: RAG検索で取得したナレッジチャンク群
- `prompt_template`: テンプレートマスターから取得した指示文
- `user_query`: ユーザー入力 + コンテンツコンテキスト + Web検索結果
- `situation_prompt`: 時間・状況設定

### テンプレート選択
プラクティスJSONの各チェインの`SETTING.PROMPT_TEMPLATE`でテンプレートコードを指定する。`"No Template"`を指定するとテンプレートなしで実行される。

## 設定場所
- マスターファイル: `user/common/mst/sample_prompt_templates.json`
- プラクティスJSON: `CHAINS[].SETTING.PROMPT_TEMPLATE`
- 環境変数: `PROMPT_TEMPLATE_MST_FILE`

## 設定例
```json
{
  "PROMPT_TEMPLATE": {
    "No Template": "",
    "RAG Query Generator": "以下の質問に対して、RAG検索用のクエリを生成してください。",
    "Dialog Digest": "以下の会話履歴を要約してください。"
  }
}
```

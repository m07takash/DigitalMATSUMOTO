# User Memory Explorer（ユーザー理解の分析・対話）

## 概要

ユーザーメモリ（Persona/Nowaday/History）を「ユーザー理解」の観点で分析するWebUI画面。サイドバーのラジオ（Chat / Knowledge Explorer / **User Memory Explorer** / Scheduler）で切替。左から **「マイメモリ」「ユーザー理解(個人)」「グループ理解」** の3タブ構成。「ユーザー理解(個人)」「グループ理解」は読み取り専用の分析＋そのデータを文脈注入した「メモリ接地エージェント」との対話、「マイメモリ」はログインユーザー自身のメモリの確認・修正。バックエンドは `DigiM_UserMemoryExplorer.py`。

## 仕組み

### タブ「マイメモリ」（自分のメモリのみ確認・修正）

**ログインユーザー自身**のメモリのみを対象とする編集タブ（他2タブが読み取り専用なのに対し、ここは書き込み可。他ユーザーは編集不可）。`st.session_state.user_id` を対象に `DigiM_UserMemory.upsert` で保存。

- **Persona**: role / summary_text の編集、各リスト項目（label テキスト上書き + status `approved/pending/deleted` 選択）、Big5（特性ごとに score 0-1 + status）。「Persona を保存」で一括 upsert（`last_reviewed` 更新）
- **Nowaday**: スナップショットを選択（`period @ 生成時刻`、最新が上）→ summary_text、継続/新規/減退/変化（1行1項目）、基本8感情（強度0-1）、二次感情（multiselect）を編集して保存。Nowadayは生成ごとに追記される履歴方式で、編集は選択した1スナップショットのみ更新（id一致upsert）
- **History**: セッションを選択 → topic / excerpt / 感情(プルチック multiselect) / confidence / active(オフで一覧・コンテキストから除外) を編集して保存

編集はウィジェットにバッファリングし、各層の Save ボタン押下時にのみ該当レコードを upsert（キー項目 service_id/user_id/session_id/period は保持）。

### タブ「ユーザー理解(個人)」（個人の深掘り）

ユーザーを1人選択し、3層を縦に可視化:

- **Persona**: 役割・要約・Big5レーダー（approved+pending、deletedのみ除外）＋右に「特性／スコア」表（statusは非表示）・要レビュー(pending)項目一覧
- **Nowaday**（最新スナップショット）: 期間要約・基本8感情レーダー＋右に「感情／スコア」表・二次感情・継続/新規/減退/変化
- **History 感情トラジェクトリ**: 期間指定（`st.date_input`レンジ、既定=現在日〜過去30日）でHistory日付を絞り込み、セッションごとのプルチック感情を日付で積み上げ（area_chart）+ セッション別感情ログ
- **Chat with this User Twin**: 選択ユーザーの記憶だけを持つAI（=そのユーザーのデジタルツイン）と対話。下記「Chat with this User Twin の実装」参照

### タブ「グループ理解」（集団の横断分析）

**対象ユーザーを multiselect で選択**（既定=全員）するのみ。関心キーワード/感情/Big5の絞り込みは廃止。

- **Persona**
  - Big5レーダー（**最大/平均/最小**の3系列を重畳）＋右に max/mean/min テーブル
  - 全ユーザーPersonaテキストのワードクラウド（MeCab名詞頻度）
  - クラスタリング（クラスタ数指定 → 埋め込み→PCA→K-Means）。表: ユーザー / クラスタ / X1 / X2 / Big5。**クラスタ解説**（`agent_23DataAnalyst.json` + 「Cluster Analyst」テンプレ、Knowledge Explorerと同様）
- **Nowaday**
  - 基本8感情レーダー（最大/平均/最小）＋右に max/mean/min テーブル
  - 二次感情ランキング（合計件数の表）
  - サマリー/継続/新規/減退/変化の **5種ワードクラウド**
  - クラスタリング（同上）＋データ表＋クラスタ解説
- **History 感情トラジェクトリ（合計）**: 対象ユーザー全Historyのプルチック感情を合計カウントしてバー表示
- **Chat with this Group Twin**: 下記参照

### Chat with this User Twin の実装（ユーザー理解(個人)タブ）

選択ユーザーのメモリだけで構成された「デジタルツイン」と対話する（`_um_twin_chat`）。

- **エンジン選択**: エージェントは選ばせない。サイドバーで選択中のエージェントに搭載された **LLMエンジン** のみ選択（`dma.get_engine_list`、既定はそのエージェントの `ENGINE.LLM.DEFAULT`）
- **文脈**: `DigiM_UserMemoryBuilder.build_context_text(service_id, user_id, [history,nowaday,persona], query_text=質問)` で都度合成。**Historyは質問文のキーワードでスコアリング**して選定（タグ×時間ハイブリッド）
- **LLM単体実行**: `DigiM_FoundationModel.call_function_by_name(engine.FUNC_NAME, 質問, system_prompt, engine, …, stream=False)` を直接呼ぶ。サイドバーのエージェントの人格・知識・システムプロンプトは一切使わない
- **振る舞い**: system_prompt で「あなたは『<ユーザー名>』本人。この記憶のみに基づき一人称で回答、記憶に無いことは『記憶にない』」と指示。AIの表示名はユーザー名（`<user_id> (User Twin / <engine>)`）
- 専用セッションは作らない（純LLM呼び出し）。対話履歴は session_state `_ume_deep_hist` に保持、サイドバー **Clear Dialogue** でクリア。直近質問で合成した文脈は expander で確認可能

### Chat with this Group Twin の実装（グループ理解タブ）

選択した集団を代表するAIと対話する（`_um_group_twin`）。

- **システムプロンプト生成**: 「生成/再生成」ボタンで対象集団のPersona/Nowaday要約からLLMが集団代表のシステムプロンプトを生成（`ux.build_group_twin_prompt`）。**text_area で手動修正可能**
- **統計ブロック自動付与**: Big5平均・基本感情平均・二次感情Top5 を system_prompt 末尾に自動付加（expanderで内容確認可）
- **エンジン選択**: User Twin と同じくサイドバー選択中エージェントの LLMエンジンのみ選択
- **LLM単体実行**: `DigiM_FoundationModel.call_function_by_name` を直接呼ぶ。専用セッションは作らない。対話履歴は `_ume_cross_g_hist`、サイドバー **Clear Dialogue** でクリア

### バックエンド主要関数（`DigiM_UserMemoryExplorer.py`）

- `list_users` / `load_bundle`
- グループ横断: `cohort_big5_stats` / `cohort_basic_emotion_stats`（max/mean/min）/ `agg_secondary_emotions` / `persona_text` / `nowaday_text` / `nowaday_field_corpus` / `cohort_history_emotion_totals` / `word_freq`（MeCab名詞頻度）/ `cluster_users`（埋め込み→PCA→K-Means）/ `explain_clusters`（LLMクラスタ解説）
- 個人: `user_emotion_trajectory` / `user_nowaday_emotion_series` / `persona_review_items`
- 対話文脈: 個人=`DigiM_UserMemoryBuilder.build_context_text`（質問でHistoryスコアリング）/ 集団=`build_group_twin_prompt`（LLM生成・手動修正可）+ 統計ブロック

### Export Report / 保存セッション

タブ群の末尾に **Export Report** セクションを配置。Knowledge Explorerと同じ仕組み:

- **Generate Report** ボタン: 現在のタブ状態（深掘りの選択ユーザー＋Persona/Nowaday/履歴トラジェクトリ＋User Twin対話、グループ理解の対象ユーザー＋Big5/感情max-mean-min表＋二次感情ランキング＋クラスタリング結果と解説＋Group Twin対話）を Markdown に集約し、`user/common/analytics/user_memory_explorer/analyticsYYYYMMDD_HHMMSS/` に `meta.json` / `state.pkl` / `report.md` として保存
- **Download (.md)**: 生成済みレポートをファイル名 `UserMemoryExplorer_<timestamp>.md` でダウンロード
- **サイドバー**: User Memory Explorer 表示中、過去の保存セッションを最新10件まで一覧表示。クリックで `state.pkl` から session_state を復元（選択ユーザー・コホート・クラスタリング結果・対話履歴等が戻る）

## 設定場所

- バックエンド: `DigiM_UserMemoryExplorer.py`
- WebUI: `WebDigiMatsuAgent.py` / `WebDigiMatsuAgent_modified.py` の `_user_memory_explorer()`
- 権限: `users.json` の `Allowed["User Memory Explorer"]`（既定 `false`、Adminは常時可）。`User Memory`（保存権限）とは独立
- データ: `DigiM_UserMemory.load_all` の3層を参照（読み取り専用、メモリ自体は変更しない）

## 設定例

```json
"RealMatsumoto": {
  "Allowed": {
    "User Memory": true,
    "User Memory Explorer": true
  }
}
```

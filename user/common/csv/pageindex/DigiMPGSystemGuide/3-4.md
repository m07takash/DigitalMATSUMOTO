# RAG知識コンテキストの構築

## 概要
ChromaDBから取得したRAGチャンクデータを、テンプレートに基づいてLLMに渡すコンテキスト文字列に変換する機能。類似度ランキング、タイムスタンプ付与、テキスト上限管理を行う。

## 仕組み

### 全体フロー（`create_rag_context`）
エージェントの`KNOWLEDGE`配列内の各RAGセット単位で処理する。

1. `DATA_TYPE: "DB"`の各データソースを`_process_rag_data()`で並列クエリ（コレクション単位で並列化）。
2. 同一IDの重複を類似度の近い方で排除。
3. `RETRIEVER`に応じて処理を分岐:
   - **Vector**（デフォルト）: `select_rag_vector()`でChromaDBからベクトル類似度検索
   - **PageIndex**: `select_rag_page_index()`でLLMがページインデックスから関連ページを選択
4. 最後に「これらの情報を踏まえて、次の質問に日本語で回答してください。」を付加。

### PageIndexによるコンテキスト取得（`select_rag_page_index`）

ベクトル検索を使わず、LLMがページの目次情報を基に関連ページを選択する方式。以下の手順で動作する。

1. `DATA`配列内の`DATA_TYPE: "PAGE_INDEX"`エントリから`DATA_NAME`を取得し、対応するページフォルダの`_index.json`を読み込む
2. インデックスの`PAGES`リスト（各ページの`id`/`title`/`summary`/`tags`/`category`）をサポートエージェント（デフォルト: `agent_59PageIndexSearch.json`）に渡す
3. LLMがクエリに関連するページIDを`MAX_PAGES`件（デフォルト5件）まで選択
4. 選択されたページIDに対応する`.md`ファイルを読み込み、コンテキスト文字列として結合
5. `HEADER_TEMPLATE`を先頭に付与し、`TEXT_LIMITS`（デフォルト6000文字）でカット
6. `LOG_TEMPLATE`で各ページの参照ログを整形（`rag_name`/`page_id`/`title`/`category`/`summary`のプレースホルダ対応）

### コンテキスト組み立て（`select_rag_vector`）
1. **TIMESTAMP設定**: `TIMESTAMP`の値に応じてチャンクの日付を決定。
   - `CREATE_DATE`: チャンクの`create_date`を使用
   - `CURRENT_DATE`: 現在日付
   - その他: 固定日付文字列
2. **TIMESTAMP_STYLE**: `strftime`形式でタイムスタンプを整形（例: `%Y年%-m月%-d日`）。
3. **CHUNK_TEMPLATE**: 各チャンクのコンテキスト文字列を生成。`{}`内のプレースホルダにチャンクデータのフィールドを展開する（例: `{value_text}`, `{timestamp}`, `{similarity_prompt}`）。
4. **類似度ソート**: `similarity_prompt`の昇順（距離が近い順）でソート。
5. **TEXT_LIMITS**: `HEADER_TEMPLATE`の文字数からカウントし、チャンクを追加。累計文字数が`TEXT_LIMITS`（+バッファ100文字）を超えたら打ち切る。
6. **LOG_TEMPLATE**: 参照情報の画面表示・記録用フォーマット。同様にプレースホルダを展開する。

### テンプレートの役割

| テンプレート | 用途 |
|---|---|
| `HEADER_TEMPLATE` | RAGコンテキストの先頭に付与するヘッダー文 |
| `CHUNK_TEMPLATE` | 各チャンクのコンテキスト本文フォーマット |
| `LOG_TEMPLATE` | 参照ログ用の記録フォーマット |

## 設定場所
- RAGセット定義: エージェントJSON の `KNOWLEDGE[]`
- テンプレートキー: `HEADER_TEMPLATE`, `CHUNK_TEMPLATE`, `LOG_TEMPLATE`
- 時刻設定: `TIMESTAMP`, `TIMESTAMP_STYLE`
- 上限設定: `TEXT_LIMITS`

## 設定例

### Vector型の設定例
```json
{
  "RAG_NAME": "Experience",
  "RETRIEVER": "Vector",
  "TIMESTAMP": "CREATE_DATE",
  "TIMESTAMP_STYLE": "%Y年%-m月%-d日",
  "HEADER_TEMPLATE": "以下はこれまでの学びです。\n",
  "CHUNK_TEMPLATE": "・({timestamp}){value_text}\n",
  "LOG_TEMPLATE": "'rag':'{rag_name}', 'ID':'{id}', 'similarity_Q':{similarity_prompt}",
  "TEXT_LIMITS": 2000
}
```

### PageIndex型の設定例
```json
{
  "RAG_NAME": "SystemGuide",
  "RETRIEVER": "PageIndex",
  "MAX_PAGES": 5,
  "HEADER_TEMPLATE": "以下は参考ドキュメントの内容です。\n",
  "LOG_TEMPLATE": "'rag':'{rag_name}', 'page_id':'{page_id}', 'title':'{title}', 'category':'{category}'",
  "TEXT_LIMITS": 6000,
  "DATA": [
    {
      "DATA_TYPE": "PAGE_INDEX",
      "DATA_NAME": "DigiMPGSystemGuide",
      "SUPPORT_AGENT": "agent_59PageIndexSearch.json"
    }
  ]
}
```

# REST API（FastAPI）

## 概要

`DigiM_API.py`で実装されたFastAPI RESTエンドポイント群。外部システムからのチャット実行、セッション参照、エージェント情報取得、フィードバック送信を提供する。

## 仕組み

**エンドポイント一覧:**

| メソッド | パス | 機能 |
|---------|------|------|
| POST | `/run` | チャット実行（メイン） |
| POST | `/run_function` | 後方互換（/runへ転送） |
| GET | `/agents` | エージェント一覧取得 |
| GET | `/agents/{agent_file}/engines` | エンジン一覧取得（LLM/IMAGEGEN） |
| GET | `/agents/{agent_file}/feedback` | フィードバック設定取得 |
| GET | `/web_search_engines` | Web検索エンジン一覧 |
| POST | `/feedback` | フィードバック送信 |
| GET | `/sessions` | セッション一覧（user_id/service_idでフィルタ可） |
| GET | `/sessions/{session_id}` | セッション履歴取得 |
| GET | `/health` | ヘルスチェック |

**POST /run のInputDataモデル:**

必須: `service_info`(SERVICE_ID, SERVICE_DATA), `user_info`(USER_ID, USER_DATA), `user_input`。任意: `session_id`, `session_name`, `situation`, `agent_file`, `engine`, `stream_mode`, `save_digest`, `memory_use`, `magic_word_use`, `meta_search`, `rag_query_gene`, `web_search`, `web_search_engine`, `private_mode`, `thinking_mode`, `user_memory`(bool: true=全層ON/false=全Off/未指定=ユーザーマスタ既定), `user_memory_layers`(["persona","nowaday","history"]サブセット, 指定時は`user_memory`より優先), `execution`(dict)

ユーザーメモリが有効な場合、対話相手のPersona/Nowaday/Historyが応答プロンプトに加えRAGクエリ生成入力にも含まれる（`memory_use=true`かつLLM実行時）。

**ロック制御:** セッション実行中はステータスが"LOCKED"。最大60秒待機（2秒間隔ポーリング）し、超過時は429を返す。

## 設定場所

- API本体: `DigiM_API.py`
- system.env: `API_AGENT_FILE`, `API_PORT`, `API_DEFAULT_SESSION_NAME`

## 設定例

```bash
# リクエスト例
curl -X POST http://localhost:8899/run \
  -H "Content-Type: application/json" \
  -d '{
    "service_info": {"SERVICE_ID": "SVC01"},
    "user_info": {"USER_ID": "user01"},
    "user_input": "AIガバナンスについて教えて",
    "thinking_mode": true
  }'

# レスポンス例
{"session_id": "API20260410...", "session_name": "AIガバナンスの解説", "response": "..."}

# ヘルスチェック
curl http://localhost:8899/health
# {"status": "ok"}
```

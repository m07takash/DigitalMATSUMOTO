# REST API（FastAPI）

## 概要

`DigiM_API.py` で実装された FastAPI RESTエンドポイント群。外部システム（LINE / Slack 等）からのチャット実行、セッション参照、エージェント・エンジン情報取得、フィードバック送信を提供する。LLM 実行は同期1リクエスト1完結（最大60秒のセッションロック待機、超過時は429）。

## 仕組み

### エンドポイント一覧

| メソッド | パス | 機能 |
|---------|------|------|
| POST | `/run` | チャット実行（メイン、同期） |
| POST | `/run_function` | 後方互換（`/run` へ転送） |
| GET | `/sessions` | セッション一覧（`user_id` / `service_id` でフィルタ可） |
| GET | `/sessions/{session_id}` | セッション履歴取得 |
| GET | `/agents` | エージェント一覧 |
| GET | `/agents/{agent_file}/engines` | LLM/IMAGEGEN エンジン一覧（default含む） |
| GET | `/agents/{agent_file}/feedback` | フィードバック設定取得 |
| POST | `/feedback` | フィードバック送信（CSV/Notion/RDBに保存。`DigiM_GeneFeedback`） |
| GET | `/web_search_engines` | Web検索エンジン一覧（default含む） |
| GET | `/health` | ヘルスチェック |

### POST /run のInputDataモデル

**必須**: `service_info`（`SERVICE_ID` / `SERVICE_DATA`）、`user_info`（`USER_ID` / `USER_DATA`）、`user_input`

**任意**: `session_id`（未指定なら `API<日時>` 発番）、`session_name`（未指定なら `gene_session_name` で `(User:<USER_ID>)<名前>` を自動生成）、`situation`、`agent_file`（既定: `API_AGENT_FILE`）、`engine`、`stream_mode`、`save_digest`、`memory_use`、`magic_word_use`、`meta_search`、`rag_query_gene`、`web_search`、`web_search_engine`（既定 `"OpenAI"`）、`private_mode`、`thinking_mode`、`user_memory`、`user_memory_layers`、`execution`（dict）

**API用デフォルト**: STREAM_MODE=true / SAVE_DIGEST=true / MEMORY_USE=true / MEMORY_SIMILARITY=false / MAGIC_WORD_USE=false / META_SEARCH=true / RAG_QUERY_GENE=true / WEB_SEARCH=false / WEB_SEARCH_ENGINE="OpenAI" / PRIVATE_MODE=false。

### ユーザーメモリの制御

優先順は **`user_memory_layers` > `user_memory` > ユーザーマスタ既定**。

- `user_memory_layers`: `["persona","nowaday","history"]` のサブセット。`[]` で全Off。指定すると最優先で `execution["USER_MEMORY_LAYERS"]` にセット。不正な層名は無視
- `user_memory`: `true`=全層ON / `false`=全Off / 未指定= `users.json` の `Allowed["User Memory Layers"]`（無ければ `USER_MEMORY_DEFAULT_LAYERS`）に従う
- `memory_use=true` かつ LLM 実行時のみ注入。ユーザーメモリテキストはRAG検索クエリ生成の入力にも含まれる

### ロック制御
セッション実行中はステータスが `LOCKED`。最大 `LOCK_WAIT_MAX=60` 秒待機（`LOCK_POLL_INTERVAL=2` 秒間隔ポーリング）し、超過時は HTTP 429 を返す。

### POST /feedback
`session_id` / `agent_file` / `seq` / `sub_seq` / `feedbacks` を受け取り、対象 seq が当該エージェントで実行されたものか、フィードバック項目が `agent.feedback.FEEDBACK_ITEM_LIST` に含まれるかを検証した上で `DigiM_GeneFeedback.create_feedback_data` で保存（旧 `COMMUNICATION` → `FEEDBACK` に統合済）。

## 設定場所

- API本体: `DigiM_API.py`
- system.env: `API_AGENT_FILE`（既定エージェント、例 `agent_10Sample.json`）、`API_PORT`（既定 8899）、`API_DEFAULT_SESSION_NAME`
- 既定Web検索エンジン: `setting.yaml` の `WEB_SEARCH_DEFAULT`

## 設定例

```bash
# ヘルスチェック
curl -s http://localhost:8899/health
# {"status": "ok"}

# メッセージ送信（user_memory層を限定）
curl -X POST http://localhost:8899/run \
  -H "Content-Type: application/json" \
  -d '{
    "service_info": {"SERVICE_ID": "API_TEST", "SERVICE_DATA": {}},
    "user_info": {"USER_ID": "TestUser", "USER_DATA": {}},
    "user_input": "AIガバナンスについて教えて",
    "agent_file": "agent_10Sample.json",
    "thinking_mode": true,
    "user_memory_layers": ["persona", "history"]
  }'

# エージェントの選択可能エンジン一覧
curl -s http://localhost:8899/agents/agent_10Sample.json/engines

# フィードバック送信
curl -X POST http://localhost:8899/feedback \
  -H "Content-Type: application/json" \
  -d '{"session_id":"API_TEST_001","agent_file":"agent_10Sample.json",
       "seq":"1","sub_seq":"1",
       "feedbacks":{"name":"Feedback","memo":{"visible":true,"flg":true,"memo":"参考になった","category":"AI"}}}'
```

# 4-6. 知識活用分析とエージェント比較

## 概要
RAGナレッジの活用度を定量的に分析する機能と、サポートエージェントのエンジン間パフォーマンスを比較評価する仕組みである。

## 仕組み

### 知識活用分析（DigiM_VAnalytics.py）
`analytics_knowledge`関数が中心処理を担う。

1. **similarity_Q vs similarity_A**: 質問ベクトルとナレッジの類似度（similarity_Q）と、回答ベクトルとナレッジの類似度（similarity_A）を比較する。差分（`knowledge_utility = similarity_Q - similarity_A`）が知識活用度を示す。
2. **正規化モード**: `ak_mode`で正規化方式を選択。`Default`（正規化なし）、`Norm(All)`（全体最大値で正規化）、`Norm(Group)`（RAGグループ内最大値で正規化）の3種。
3. **統計量算出**: RAGごとにsimilarity_Q/A/knowledge_utilityのmin/mean/median/max/varianceを計算する。
4. **可視化**:
   - 知識活用プロット: similarity_QとAの横棒グラフ（`create_similarity_plot_file`）。クエリ種別（NORMAL/INTENT）で色分け。
   - PCA散布図: ナレッジベクトルを2次元に圧縮し、参照有無で色分け。説明率をタイトルに表示。
   - t-SNE散布図: 非線形次元削減による可視化。perplexityパラメータを自動調整。
   - カテゴリ散布図: `category_map.json`で定義されたカテゴリ色で分類表示。

### エージェント比較（DigiM_SupportEval.py）
サポートエージェントの各エンジンを同一質問セットで比較評価するベンチマークツール。

1. **評価対象**: `intent`（RAGクエリ生成）と`meta`（メタデータ検索）の2種。
2. **エンジン切り替え**: `override_engine`で`ENGINE.LLM.DEFAULT`を一時的に差し替え、各エンジンで質問を実行する。
3. **評価指標**: 応答時間（avg/min/max）、成功/エラー件数、プロンプト/レスポンストークン数を記録。
4. **出力**: Excel形式で詳細結果・ピボットテーブル・サマリーを出力。WebUI用のバイト列返却にも対応。
5. **CLI実行**: `python3 DigiM_SupportEval.py input.xlsx`で実行可能。`--target`でintent/metaを選択。

## 設定場所
- 知識活用分析: Web UIの分析画面から実行
- エージェント比較: CLI or Web UIの評価画面
- カテゴリマップ: `user/common/mst/category_map.json`

## 設定例
```python
# 知識活用分析の実行
analytics_knowledge(
    agent_file="agent_01DigitalMATSUMOTO.json",
    ref_timestamp="2025-01-01 00:00:00.000",
    title="分析タイトル",
    reference=knowledge_ref_data,
    analytics_file_path="output/",
    ak_mode="Default",
    dim_mode={"method": "PCA", "params": {}}
)
```

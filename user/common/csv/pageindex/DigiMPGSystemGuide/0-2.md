# 実行フロー（リクエスト処理の流れ）

## 概要

`DigiM_Execute.py`の`DigiMatsuExecute()`関数が全処理パイプラインを制御する。ユーザー入力からLLMレスポンス生成、履歴保存までを一貫して実行する。

## 仕組み

処理は以下のステップで進行する（タイムスタンプログで追跡可能）。

1. **セッション設定 [01]**: `DigiMSession`を宣言し、シーケンス番号を採番
2. **エージェント設定 [02]**: `DigiM_Agent`を初期化。`overwrite_items`があればエージェント設定を上書き
3. **コンテンツ読込 [03]**: 添付ファイル（PDF/画像/音声等）からコンテキストを抽出
4. **シチュエーション設定 [04]**: 日時・状況の指示をプロンプトに付加
5. **Thinking Mode判定 [04.5]**（任意）: `_run_thinking_agent()`でユーザークエリをAIが分析し、使用するHabit・Web検索・RAG戦略・Bookを動的に選択。結果が空の場合はMagic Word判定にフォールバック
6. **会話ダイジェスト取得 [05]**: 過去の会話のダイジェスト（要約）を取得
7. **Web検索 [06]**: Perplexity/OpenAI/Google経由でWeb検索を実行（任意）
7. **クエリベクトル化 [07]**: `embed_texts_batch()`でバッチ埋め込み
8. **並列実行 [08-10]**: `ThreadPoolExecutor(max_workers=3)`で以下を同時実行
   - 会話メモリ取得（トークン制限付き）
   - RAGクエリ生成（意図分析LLM）
   - メタデータ検索（日付抽出LLM）
9. **RAGコンテキスト取得 [11]**: `RETRIEVER`の種類に応じて分岐。`Vector`の場合はChromaDBからベクトル検索、`PageIndex`の場合はLLMがページインデックスから関連ページを選択してナレッジを取得
10. **プロンプト構築 [12]**: テンプレート + ナレッジ + ユーザークエリ + シチュエーションを結合
11. **LLM実行 [13]**: ストリーミングモードでLLMを呼び出し、逐次レスポンスをyield
12. **類似度評価 [15]**: レスポンスとメモリ・ナレッジ間の類似度を算出
13. **ログ一括保存 [17]**: `save_history_batch()`で会話履歴をJSON保存
14. **ダイジェスト生成（バックグラウンド）**: 別スレッドでダイジェストを生成し、完了後にセッションをUNLOCK

## 設定場所

- 実行設定: `execution`辞書（`STREAM_MODE`, `WEB_SEARCH`, `RAG_QUERY_GENE`等）
- Web検索エンジン: `setting.yaml`の`WEB_SEARCH_DEFAULT`
- メモリ設定: エージェントJSON内`ENGINE.LLM.MEMORY`

## 設定例

```python
execution = {
    "STREAM_MODE": True,
    "WEB_SEARCH": True,
    "WEB_SEARCH_ENGINE": "Perplexity",
    "RAG_QUERY_GENE": True,
    "META_SEARCH": True,
    "THINKING_MODE": False,
}
```

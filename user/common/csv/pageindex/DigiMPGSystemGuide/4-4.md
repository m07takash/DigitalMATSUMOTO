# 4-4. サポートエージェント群

## 概要
メインのLLM実行を補助する専用エージェント群である。RAGクエリ生成、日付抽出、ダイジェスト生成、セッション名生成などの役割を持ち、ThreadPoolExecutorによる並列実行で処理を高速化する。

## 仕組み

### 各サポートエージェントの役割

1. **RAG_QUERY_GENERATOR** (`DigiM_Tool.py: RAG_query_generator`): ユーザーの質問からRAG検索に最適なクエリを生成する。メモリ・シチュエーション・クエリベクトルを入力として受け取り、検索意図を明確化した追加クエリとそのベクトルを返す。デフォルトエージェント: `agent_56RAGQueryGenerator.json`

2. **EXTRACT_DATE** (`DigiM_Tool.py: extract_date`): ユーザーの質問からメタデータ検索用の日付情報を抽出する。LLMが日付を解析し、`extract_list_pattern`と`merge_periods`で日付範囲リストに変換する。デフォルトエージェント: `agent_55ExtractDate.json`

3. **DIALOG_DIGEST** (`DigiM_Tool.py: dialog_digest`): 会話メモリを要約してダイジェストを生成する。メモリのuser/assistant発話をテキスト化し、LLMで圧縮要約する。出力は「これまでの会話のダイジェスト」として保存される。デフォルトエージェント: `agent_51DialogDigest.json`

4. **gene_session_name** (`DigiM_Tool.py: gene_session_name`): 会話内容からセッション名を自動生成する。メモリとユーザークエリを入力としてLLMが短いタイトルを生成する。デフォルトエージェント: `agent_57SessionName.json`

5. **PAGE_INDEX_SEARCH** (`DigiM_Tool.py: page_index_search`): PageIndex型RAGでページ選択を行うサポートエージェント。ページインデックス（`_index.json`のPAGESリスト）とユーザークエリを受け取り、LLMが関連度の高いページIDを`MAX_PAGES`件まで選択して返す。`select_rag_page_index()`から呼び出される。デフォルトエージェント: `agent_59PageIndexSearch.json`

6. **PERSONA_SELECTOR** (`agent_54PersonaSelector.json`, Phase 7): `chain.PERSONAS="THINKING"` のステップで、選択中ORG配下のペルソナ候補プールから、ユーザー質問に最適なペルソナを **最大N人** 自動選定する。1人で十分なら1人、専門外は除外。失敗・0人選定時はUI multiselectの選択にフォールバック。選定理由は `_THINKING_RESULT.personas_reason` に記録。上限N は WebUI **Max Personas** 入力（既定は `setting.yaml` の `MAX_PERSONAS=3`）。エージェントJSONの `SUPPORT_AGENT.PERSONA_SELECTOR` で差し替え可能。

7. **PERSONA_MERGE** (`agent_50PersonaMerge.json`, Phase 6): `chain.PERSONA_MERGE="summary"` のときに呼ばれる統合用LLM。`DigiM_Tool.dialog_persona_merge()` から呼び出され、各ペルソナ応答を要約・統合する。プロンプト「Persona Merge」の `{summary_level}` プレースホルダで要約強度を制御（`light` / `medium` / `heavy` または自由記述。`chain.PERSONA_MERGE_LEVEL` で指定）。

### 並列実行
`DigiM_Execute.py`のステップ08-10で`ThreadPoolExecutor(max_workers=3)`を使い、以下を同時実行する。

- メモリ取得 (`session.get_memory`)
- RAGクエリ生成 (`_build_intent_queries`)
- メタデータ検索 (`_build_meta_searches`)

各Futureの結果を`result()`で回収し、後続のRAG検索に利用する。

### ダイジェスト生成のバックグラウンド化
`_run_digest_background`関数で`threading.Thread(daemon=True)`として起動され、メインの応答フローをブロックしない。

## 設定場所
エージェントJSONの`SUPPORT_AGENT`セクション

## 設定例
```json
{
  "SUPPORT_AGENT": {
    "RAG_QUERY_GENERATOR": "agent_56RAGQueryGenerator.json",
    "EXTRACT_DATE": "agent_55ExtractDate.json",
    "DIALOG_DIGEST": "agent_51DialogDigest.json",
    "ART_CRITICS": "agent_52ArtCritic.json",
    "THINKING": "agent_70DigiMThinking.json"
  }
}
```

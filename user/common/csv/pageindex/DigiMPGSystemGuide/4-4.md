# 4-4. サポートエージェント群

## 概要
メインのLLM実行を補助する専用エージェント群である。RAGクエリ生成、日付抽出、ダイジェスト生成、セッション名生成、PageIndex選択、Thinking判定、Persona選定/マージ、画像批評、テキスト比較などの役割を持ち、ThreadPoolExecutorによる並列実行で処理を高速化する。

## 仕組み

### 各サポートエージェントの役割

1. **RAG_QUERY_GENERATOR** (`DigiM_Tool.RAG_query_generator`): ユーザー質問からRAG検索に最適なクエリを生成。メモリ・シチュエーション・クエリベクトルを入力し、検索意図を明確化した追加クエリとベクトルを返す。ユーザーメモリ有効時はメモリ文脈も入力に含まれる。既定: `agent_56RAGQueryGenerator.json`

2. **EXTRACT_DATE** (`DigiM_Tool.extract_date`): メタデータ検索用の日付情報を抽出。LLMが日付を解析し`merge_periods`で日付範囲リストへ変換。既定: `agent_55ExtractDate.json`

3. **DIALOG_DIGEST** (`DigiM_Tool.dialog_digest`): 会話メモリを要約してダイジェストを生成。バックグラウンドスレッド（`_run_digest_background`）で実行され、最終チェイン完了時にUNLOCK。既定: `agent_51DialogDigest.json`

4. **SESSION_NAME** (`DigiM_Tool.gene_session_name`): 会話内容からセッション名を自動生成。既定: `agent_57SessionName.json`

5. **PAGE_INDEX_SEARCH** (`DigiM_Tool.page_index_search`): PageIndex型RAGでページ選択を行うサポートエージェント。`_index.json`のPAGESリストとユーザークエリを受け、`MAX_PAGES`件まで関連ページを返す。`select_rag_page_index()`から呼び出され、`BOOK[].DATA[].SUPPORT_AGENT` で個別指定可能。既定: `agent_59PageIndexSearch.json`

6. **THINKING** (`DigiM_Tool.thinking_agent`): Thinking Mode時にHabit・Web検索・RAGクエリ・Books等を自動判定。既定: `agent_58Thinking.json`（Digital MATSUMOTO本体: `agent_70DigiMThinking.json`）

7. **PERSONA_SELECTOR** (`agent_54PersonaSelector.json`): `chain.PERSONAS="THINKING"` のステップで、選択中ORG配下のペルソナプールから最適なペルソナを **最大N人** 自動選定。1人で十分なら1人、専門外は除外。失敗・0人時はUI multiselectにフォールバック。選定理由は `_THINKING_RESULT.personas_reason` に記録。上限N は WebUI **Max Personas**（既定 `setting.yaml` の `MAX_PERSONAS=3`）。

8. **PERSONA_MERGE** (`agent_50PersonaMerge.json`): `chain.PERSONA_MERGE="summary"` 時に呼ばれる統合用LLM。`DigiM_Tool.dialog_persona_merge()` から呼び出され、各ペルソナ応答を要約・統合する。プロンプト「Persona Merge」の `{summary_level}` で要約強度を制御。

9. **ART_CRITICS** (`DigiM_Tool.art_critics`): 画像を批評。既定: `agent_52ArtCritic.json`。詳細は 4-5。

10. **COMPARE_TEXTS** (`DigiM_Tool.compare_texts`): 2テキストの比較。`agent_53CompareTexts.json` 固定。詳細は 4-5。

11. **DATA_ANALYST** (`agent_23DataAnalyst.json`): Knowledge Explorer の Cluster / Trend / Topic 解説、および User Memory Explorer のクラスタ解説 / Group Twin プロンプト生成で利用される汎用解析LLM（4-6 / 5-5）。

12. **ユーザーメモリ生成** (`agent_62UserMemoryHistory.json` / `agent_61UserMemoryNowaday.json` / `agent_60UserMemoryPersona.json`): History/Nowaday/Persona 各層のLLM抽出。Practice (`practice_13/14/15UserMemory*.json`) 経由で呼ばれる（3-10）。

サンプル用途として `agent_21EthicalCheck.json` / `agent_22SenryuSensei.json` などの専用エージェントもプラクティスから AGENT_FILE 指定で利用される。

### 並列実行
`DigiM_Execute.py`のステップ08-10で`ThreadPoolExecutor(max_workers=3)`を使い、メモリ取得・RAGクエリ生成（`_build_intent_queries`）・メタデータ検索（`_build_meta_searches`）を同時実行する。

### ダイジェスト生成のバックグラウンド化
`_run_digest_background`関数で`threading.Thread(daemon=True)`として起動され、メインの応答フローをブロックしない。最終チェインのダイジェスト完了時のみセッションをUNLOCK（`_UNLOCK_ON_DIGEST`）。

## 設定場所
エージェントJSONの`SUPPORT_AGENT`セクション。PageIndex検索のみ `BOOK[].DATA[].SUPPORT_AGENT` で個別差し替え可。

## 設定例
```json
{
  "SUPPORT_AGENT": {
    "DIALOG_DIGEST": "agent_51DialogDigest.json",
    "ART_CRITICS": "agent_52ArtCritic.json",
    "EXTRACT_DATE": "agent_55ExtractDate.json",
    "RAG_QUERY_GENERATOR": "agent_56RAGQueryGenerator.json",
    "THINKING": "agent_58Thinking.json"
  }
}
```

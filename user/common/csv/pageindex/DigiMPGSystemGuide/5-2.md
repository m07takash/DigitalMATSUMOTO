# Detail Information・分析ダッシュボード

## 概要

各チャット応答に付随するDetail Informationエクスパンダと、Analytics Resultsエクスパンダで構成される詳細情報・分析画面。`DigiM_Session.get_detail_info()`が情報を組み立てる。

## 仕組み

**Detail Information（get_detail_info）の表示項目:**

1. **実行情報:** 実行関数名（FUNC_NAME）、プロンプトテンプレート設定、RAGデータ一覧
2. **実行結果:** エージェントファイル名、実行モデル名とパラメータ、回答時間（prompt〜response間の差分）、入力/出力トークン数
3. **実行履歴:** timestamp_log（ステップごとのログ）
4. **会話のダイジェスト:** ダイジェスト生成に使用したエージェント・モデル・**実行時間**（`timestamp_start`〜`timestamp`の差分）・出力トークン数・テキスト
5. **メモリ:** 参照されたメモリログ一覧
6. **【Thinking】:** Thinking Mode有効時に表示。エージェントファイル名、使用モデル、実行時間、入力/出力トークン数、思考内容（reasoning）、判定結果（選択されたhabit・web_search・books等）を表示
7. **RAG検索用クエリ:** クエリ生成のエージェント・モデル・実行時間・トークン数・LLMレスポンス
8. **メタ検索:** 日付検索の条件・モデル・トークン数・検索条件リスト
9. **RAGコンテキスト:** 検索されたRAGドキュメントの内容一覧
10. **【PageIndex選択ページ】:** PageIndex型RAG使用時に表示。LLMが選択したページのID・タイトル・カテゴリ・サマリーを一覧表示。`knowledge_rag`参照情報内の`page_id`を含むエントリから`parse_log_template()`でフィールドを抽出して表示する
11. **コンテンツコンテキスト:** 添付ファイルのコンテキスト
12. **WEB検索結果:** 検索エンジン名・モデル・実行時間・コンテキスト・参考URL一覧
13. **【ユーザーメモリ】:** 当該チャットでKnowledge直前に注入されたユーザーメモリの参照ID（short:.../mid:.../long:...）+ 注入コンテキスト全文を表示。`response.reference.user_memory` と `prompt.user_memory_context` から復元（詳細は3-10）

各ブロックは【】見出しで区切られ、UIではブロックごとにCopyボタンが付与される。

**Analytics Results の表示項目（チャット応答下の expander）:**

- **Knowledge Utility:** RAGの知識活用性分析（`DigiM_VAnalytics.analytics_knowledge`）。`knowledge_utility = similarity_Q - similarity_A` を表示。散布図（ref/category別）・類似度プロット・座標CSVをDataFrame表示。分析モードは Default / Norm(All) / Norm(Group)、次元削減は PCA / t-SNE。深掘りは Knowledge Explorer（4-6）へ移管
- **Compare Agents:** 別エージェント・エンジンで同一質問を実行し、応答のコサイン距離（diff）と比較テキスト（`compare_texts` / `agent_53CompareTexts.json`）を表示。比較結果ごとに Knowledge Utility 分析も可能

## 設定場所

- 詳細情報ロジック: `DigiM_Session.py` の `get_detail_info()`
- 分析ロジック: `DigiM_VAnalytics.py` の `analytics_knowledge()`
- 権限制御: ユーザーマスタの `Allowed.Details`, `Allowed.Analytics Knowledge`, `Allowed.Analytics Compare`

## 設定例

```json
{
  "Allowed": {
    "Details": true,
    "Analytics Knowledge": true,
    "Analytics Compare": true
  }
}
```

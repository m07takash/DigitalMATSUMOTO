# セッション永続化とステータス管理

## 概要

`DigiM_Session.py` の `DigiMSession` クラスがチャット履歴の保存・読込・ステータス管理を担う。JSONファイルベースの永続化とファイルパス単位のスレッドロックによる排他制御を実装している。

## 仕組み

### フォルダ構造

セッションごとに以下の構造で保存される（フォルダ名は `setting.yaml` の `SESSION_FOLDER_PREFIX` + session_id）。

```
user/session{session_id}/
├── chat_memory.json       # 会話履歴（全 seq / sub_seq）
├── status.yaml            # ステータス・メタデータ
├── contents/              # 添付ファイル・生成画像
├── vecs/                  # ベクトルデータ（.npy）
└── analytics/             # 分析結果
```

### 会話履歴の保存 (`save_history_batch`)

ファイルパス単位のロック (`_get_file_lock`) で排他制御した上で、`chat_memory.json` に対して seq / sub_seq 階層で会話データを一括書き込む。ダイジェスト生成はバックグラウンドスレッドで後追い追記する。保存後にDB Exportステータスを自動的にUNDOにリセットする。

### ファイルロック

モジュールレベルの `_session_file_locks: dict` と メタロック `_session_file_locks_meta = threading.Lock()` で、ファイルパス単位の `threading.Lock` をスレッドセーフに払い出す。並列実行（複数ペルソナ／API並走）下のRace Condition防止のため、`save_history_batch` と関連書き込みで使用される。

### ステータス管理 (`status.yaml`)

| キー | 用途 |
|------|------|
| `status` | LOCKED / UNLOCKED（実行中ロック） |
| `message` | 処理中のステータスメッセージ |
| `response` | 疑似ストリーミング用の途中レスポンス |
| `active` | Y / N（セッション有効/無効） |
| `name` | セッション名 |
| `agent` | エージェントファイル名 |
| `last_update_date` | 最終更新日時 |
| `db_export` | DB Exportステータス（DONE / UNDO） |

### セッションアーカイブ

`archive_old_sessions(days)` で `ARCHIVE_DAYS`（既定30）以上更新のないセッションフォルダを `user/archive/` 配下にZIP圧縮し、元フォルダを削除する。

## 設定場所

- フォルダ設定: `setting.yaml` の `SESSION_FOLDER_PREFIX` / `SESSION_FILE_NAME` / `SESSION_STATUS_FILE_NAME` / `SESSION_CONTENTS_FOLDER` / `SESSION_ANALYTICS_FOLDER`
- アーカイブ: `ARCHIVE_FOLDER` / `ARCHIVE_DAYS`

## 設定例

```yaml
SESSION_FOLDER_PREFIX: "session"
SESSION_FILE_NAME: "chat_memory.json"
SESSION_STATUS_FILE_NAME: "status.yaml"
ARCHIVE_FOLDER: "user/archive/"
ARCHIVE_DAYS: 30
```

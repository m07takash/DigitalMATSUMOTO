# セッション永続化とステータス管理

## 概要

`DigiM_Session.py`の`DigiMSession`クラスがチャット履歴の保存・読込・ステータス管理を担う。JSONファイルベースの永続化とファイルロックによる排他制御を実装している。

## 仕組み

### フォルダ構造

セッションごとに以下の構造で保存される。

```
user/session{session_id}/
├── chat_memory.json       # 会話履歴（全seq/sub_seq）
├── status.yaml            # ステータス・メタデータ
├── contents/              # 添付ファイル・生成画像
├── vecs/                  # ベクトルデータ（.npy）
└── analytics/             # 分析結果
```

### 会話履歴の保存 (`save_history_batch`)

ファイルロック(`_get_file_lock`)で排他制御した上で、`chat_memory.json`に対してseq/sub_seq階層で会話データを一括書き込む。ダイジェスト生成はバックグラウンドスレッドで後から追記する。保存後にDB Exportステータスを自動的にUNDOにリセットする。

### ステータス管理

`status.yaml`に以下の情報を保持する。

| キー | 用途 |
|------|------|
| `status` | LOCKED/UNLOCKED（実行中ロック） |
| `message` | 処理中のステータスメッセージ |
| `response` | 疑似ストリーミング用の途中レスポンス |
| `active` | Y/N（セッション有効/無効） |
| `name` | セッション名 |
| `agent` | エージェントファイル名 |
| `last_update_date` | 最終更新日時 |
| `db_export` | DB Exportステータス（DONE/UNDO） |

### ファイルロック

`_session_file_locks`辞書とメタロック(`_session_file_locks_meta`)でファイルパス単位のスレッドセーフなロックを管理する。Race Condition防止のためにsave_history_batchで使用する。

### セッションアーカイブ

`archive_old_sessions()`で設定日数（デフォルト30日）以上更新のないセッションをZIP圧縮し元フォルダを削除する。

## 設定場所

- フォルダ設定: `setting.yaml`の`SESSION_FOLDER_PREFIX`, `SESSION_FILE_NAME`, `SESSION_STATUS_FILE_NAME`
- アーカイブ日数: `setting.yaml`の`ARCHIVE_DAYS`

## 設定例

```yaml
SESSION_FOLDER_PREFIX: "session"
SESSION_FILE_NAME: "chat_memory.json"
SESSION_STATUS_FILE_NAME: "status.yaml"
ARCHIVE_DAYS: 30
```

# トークン計量とテキストサニタイズ

## 概要

LLMへの入出力テキストのトークン数を計算し、不正文字を除去するユーティリティ群。プロンプトのトークン制限管理とJSON/XMLパースエラー防止に使用される。

## 仕組み

### トークン計量 (`DigiM_Util.count_token`)

3種類のトークナイザーをサポートする。

| トークナイザー | 方式 | 用途 |
|-------------|------|------|
| `tiktoken` | OpenAI cl100k_base エンコーディング | GPT / Azure OpenAI / Claude / 汎用 |
| `gemini` | Gemini API `count_tokens` | Gemini専用 |
| その他 | `len(text)` 文字数 | フォールバック |

エージェントJSONの `TOKENIZER` フィールドで指定する。メモリ取得時にトークン上限(`MEMORY.limit`)と比較して会話履歴を切り取る。

### テキストサニタイズ

**DigiM_FoundationModel.py:**
- `_sanitize_text(text)`: NUL文字（`\x00`）、孤立サロゲートペア、C0制御文字（`\t`/`\n`/`\r` は保持）を除去
- `_sanitize_messages(messages)`: メッセージリスト・辞書を再帰的に走査し、全テキスト要素に `_sanitize_text()` を適用。文字列・リスト・辞書のネストに対応。**全LLM API送信直前**（GPT / Azure OpenAI / Gemini / Claude / Grok / Llama / DALL-E / Azure DALL-E など）で適用される

**DigiM_Util.py:**
- `sanitize_text(text)`: LLMレスポンスからJSON/XMLで問題になる制御文字を除去。サロゲートペア（絵文字等）は保持
- LLMレスポンス受信後に `DigiM_Execute.py` で適用

### 使い分け

入力側（プロンプト）は `_sanitize_messages()`、出力側（レスポンス）は `dmu.sanitize_text()` を使用する。

## 設定場所

- トークナイザー指定: エージェントJSON `ENGINE.*.TOKENIZER`
- メモリトークン上限: エージェントJSON `ENGINE.*.MEMORY.limit`

## 設定例

```json
{
  "TOKENIZER": "tiktoken",
  "MEMORY": {"limit": 10000}
}
```

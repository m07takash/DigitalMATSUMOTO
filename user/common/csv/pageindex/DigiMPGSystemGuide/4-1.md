# 4-1. プラクティスチェインとマルチステップ実行

## 概要
プラクティスは複数のステップ（チェイン）を順次実行するマルチステップ実行の仕組みである。LLM呼び出し、画像生成、ツール実行を組み合わせたワークフローを定義できる。

## 仕組み
`DigiM_Execute.py`の`DigiMatsuExecute_Practice`関数が中心処理を担う。

### CHAIN（ステップ配列）の構造
プラクティスJSONの`CHAIN`配列に各ステップを定義する。各ステップは以下の属性を持つ。

- **TYPE**: 実行タイプ。`LLM`（テキスト生成）、`IMAGEGEN`（画像生成）、`FUNC`（関数呼び出し）の3種。
- **AGENT_FILE / OVERWRITE_ITEMS / ADD_KNOWLEDGE / PROMPT_TEMPLATE / MEMORY_USE / USER_INPUT / CONTENTS / SITUATION**: 各ステップの詳細設定。

### ステップ間データフロー
`_resolve_user_input`関数と`_resolve_contents`関数でステップ間のデータ参照を解決する。

- `USER`: 元のユーザー入力を使用
- `INPUT_N`: ステップNの入力テキストを参照
- `OUTPUT_N`: ステップNの出力テキストを参照
- `EXPORT_N`: ステップNのエクスポートコンテンツを参照
- リテラル文字列: 固定テキストを挿入
- リスト指定: 複数要素を結合

### エージェント切り替え
各ステップの`AGENT_FILE`で別エージェントを指定可能。`USER`を指定すると呼び出し元のエージェントを使用する。

### OVERWRITE_ITEMS
`OVERWRITE_ITEMS`でエージェント設定を一時的に上書きできる。`USER`指定でUI側の設定を引き継ぎ、辞書指定でプラクティス固有の設定を適用する。

### ADD_KNOWLEDGE
`ADD_KNOWLEDGE`で追加ナレッジを指定する。`USER`を含むとHabitの`ADD_KNOWLEDGE`とUI選択のBOOKも追加される。

### チェイン間のUNLOCK制御（マルチチェインロック修正）
マルチチェイン実行時、中間チェインのダイジェスト生成バックグラウンドスレッドがセッションを早期にUNLOCKしてしまう問題が修正された。`DigiMatsuExecute_Practice`内で各チェインの実行時に`_is_last_chain`フラグを判定し、`execution["_UNLOCK_ON_DIGEST"]`に設定する。`DigiMatsuExecute`側では`execution.get("_UNLOCK_ON_DIGEST", True)`でこのフラグを参照し、ダイジェスト完了時のUNLOCK処理を制御する。これにより中間チェインのダイジェストスレッドはUNLOCKをスキップし、最終チェインのダイジェスト完了時のみセッションがUNLOCKされる。

### chain.PERSONAS（ステップ単位のマルチペルソナ並列実行・Phase 6/7）
個々のチェインに `PERSONAS` を設定すると、そのステップだけを複数ペルソナで並列実行 → マージ → 次ステップへ、というワークフローが組める。

| chain属性 | 値 | 意味 |
|----------|----|------|
| `PERSONAS` | `"WEB_UI"` | WebUIで選択中のペルソナを使用 |
|  | `["P0001","P0002",...]` | persona_id を直接指定 |
|  | `"THINKING"` | `agent_54PersonaSelector.json` が選択中ORG配下から最大 `MAX_PERSONAS` 人を自動選定（Phase 7） |
| `PERSONA_MERGE` | `summary` / `concat` / `first` / `include_query` / `none` | 並列応答の統合方式（`summary` は `agent_50PersonaMerge.json` 経由） |
| `PERSONA_MERGE_LEVEL` | `light` / `medium` / `heavy` または自由記述 | `summary` 時の要約強度（プロンプト「Persona Merge」の `{summary_level}` に展開） |

`_resolve_step_personas()` が `chain.PERSONAS` を実ペルソナdictリストに解決し、`ThreadPoolExecutor`（上限 `setting.yaml` の `MAX_PARALLEL_PERSONAS`、既定 4）で並列実行する。各ペルソナsub_seqには `setting.persona_id` / `chain_role="persona"` / `memory_flg="N"` が自動付与され、次ターンの会話メモリから除外される（画面表示は残る）。次ステップは `OUTPUT_<開始sub_seq>` でマージ結果を参照する。

詳細とサンプル（`practice_51PersonaCompare.json` / `practice_52PersonaThinking.json`）は 3-9 を参照。

## 設定場所
プラクティスJSON（`user/common/practice/`配下）

## 設定例
```json
{
  "NAME": "ImageGen",
  "CHAIN": [
    {"TYPE": "IMAGEGEN", "AGENT_FILE": "USER", "PROMPT_TEMPLATE": "Image Gen", "MEMORY_USE": true},
    {"TYPE": "LLM", "AGENT_FILE": "agent_52ArtCritic.json", "PROMPT_TEMPLATE": "Art Critic",
     "USER_INPUT": "コンテンツについて、これまでの話と関連付けながら300文字程度で解説してください。",
     "CONTENTS": "EXPORT_1", "MEMORY_USE": true}
  ]
}
```

# 3-6. シチュエーション・時間コンテキスト

## 概要
シチュエーション機能は、LLMに対して現在の状況や時間設定をプロンプトに注入する仕組みである。標準的な日時と架空の日時設定の両方に対応し、ロールプレイや時間指定シナリオを実現する。

## 仕組み
`DigiM_Execute.py`の単体実行関数内（プロンプト組み立て直前）でシチュエーションプロンプトが生成される。

1. **SITUATION辞書構造**: `situation`辞書は`TIME`キーと`SITUATION`キーを持つ。
2. **標準時間判定**: `TIME`値を`%Y/%m/%d %H:%M:%S`形式でパースし、成功すれば標準日時と判定する。標準日時の場合は「現在は{TIME}です。」という簡潔な指示になる。
3. **架空時間設定**: パースに失敗した場合（例: "西暦3000年"など）、架空の日時設定と判断し、強い指示文を生成する。「この会話では、現在の日時は{TIME}として設定されています。会話履歴やシステム上の実際の日時に関わらず、必ずこの設定に従ってください。」という文面でLLMに強制する。
4. **SITUATIONテキスト**: `SITUATION`キーにはフリーテキストで追加の状況説明を記述でき、`TIME`と結合してプロンプトに付加される。

生成された`situation_prompt`は最終プロンプトの末尾に追加される（`query = knowledge_context + prompt_template + user_query + situation_prompt`）。

## 設定場所
Web UIのシチュエーション入力欄、またはプラクティスJSONの`SETTING.SITUATION`で設定する。`USER`を指定すると呼び出し元の設定を引き継ぐ。

## 設定例
```json
{
  "SITUATION": {
    "TIME": "2025/01/01 00:00:00",
    "SITUATION": "新年の挨拶をしている場面です"
  }
}
```

架空時間の例:
```json
{
  "TIME": "江戸時代 天保年間",
  "SITUATION": "あなたは江戸の商人です"
}
```

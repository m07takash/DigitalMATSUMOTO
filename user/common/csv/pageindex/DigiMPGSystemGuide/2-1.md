# RAGデータ取り込み（CSV・Notion・Excel）

## 概要
RAGデータをCSV／Notion DB／Excelファイルから取り込み、ChromaDBまたはページインデックスとして保存する機能。`generate_rag()`がエントリーポイントとなり、`rags.json`マスターに定義された各RAG設定を順に処理する。

## 仕組み
`generate_rag()`は`rags.json`を読み込み、`active: "Y"`のエントリを対象に以下を実行する。

- **CSV入力** (`input: "csv"`): `get_chunk_csv()`を呼び出す。`field_items`で定義したCSV列名を`DictReader`で読み込み、`title`/`key_text`/`value_text`の各リストで指定した列を連結してチャンクを生成する。`category_items`による行フィルタも可能。
- **Notion入力** (`input: "notion"`): `get_chunk_notion()`または`get_chunk_notion_pageindex()`（`data_type: "pageindex"`時）を呼び出す。`item_dict`でNotionプロパティ名と型（title/date/rich_text/select/chk等）のマッピングを定義する。`chk_dict`でチェックボックス条件、`date_dict`で日付範囲、`category_dict`でセレクト条件によるフィルタリングを行う。
- **Excel入力** (`input: "excel"`, 現状 `data_type: "pageindex"` のみ): `get_chunk_excel_pageindex()`を呼び出す。1行1ページのExcelシートを`source_dir`/`source_file`/`sheet`で指定し、`item_dict`の各キー（`book`/`id`/`title`/`summary`/`tags`/`category`/`body`）にExcelの列名を割り当てる。`body`セルが`source_dir`配下の`.txt`/`.md`ファイル名と一致すればそのファイル本文を読込、一致しなければセル値自体を本文として扱う。`tags`はカンマ／縦棒区切りで自動分割。
- 保存先は`data_type`で決まる:
  - `chromadb`: `save_rag_chunk_db()`でChromaDBにベクトル埋め込みと共に保存
  - `pageindex`（新形式）: `save_rag_pageindex()`で`{bucket}/{id}.md`と`_index.json`（BOOK + PAGES）を生成。`id`の形式（例: `1-2-3`）から`sort_order`を動的算出、id重複は上書き更新。
  - `page_index`（旧形式）: `save_rag_pages()`でページ単位の.mdファイルと`_index.json`を生成・保存
- Notion入力の場合、`fin_flg`を設定すると登録完了後にNotionページのチェックボックスを自動更新する（CSV/Excel入力では無視）。

## 設定場所
- RAGマスター定義: `user/common/mst/rags.json`
- 各エントリのキー（共通）: `active`, `input`, `data_type`, `bucket`
- CSV固有: `file_path`, `file_name`, `field_items`, `title`, `key_text`, `value_text`, `category_items`
- Notion固有: `data_name`, `item_dict`, `chk_dict`, `date_dict`, `category_dict`, `fin_flg`
- Excel固有: `source_dir`, `source_file`, `sheet`, `item_dict`
- Notion DBマスター: 環境変数`NOTION_MST_FILE`で指定されたJSONファイル（MST_FOLDER内）

## 設定例

CSV → ChromaDB:
```json
{
  "Sample02_Memo": {
    "active": "Y",
    "input": "csv",
    "data_type": "chromadb",
    "bucket": "Sample02_Memo",
    "file_path": "user/common/csv/",
    "file_name": "Sample02_Memo.csv",
    "field_items": ["create_date", "memo"],
    "category_items": [],
    "title": ["create_date"],
    "key_text": ["memo"],
    "value_text": ["memo"]
  }
}
```

Excel → PageIndex:
```json
{
  "SampleConsultingBookExcel": {
    "active": "Y",
    "input": "excel",
    "data_type": "pageindex",
    "bucket": "SampleConsultingBookExcel",
    "source_dir": "user/common/csv/pageindex/SampleConsultingBookExcel",
    "source_file": "SampleConsultingBookExcel.xlsx",
    "sheet": "pages",
    "item_dict": {
      "book": "ブック名", "id": "ID", "title": "タイトル",
      "summary": "サマリー", "tags": "タグ",
      "category": "カテゴリ", "body": "本文"
    }
  }
}
```

## 関連機能: ページインデックスのローカルダウンロード
WebUI **RAG Management → Page Index Export** から、既存ページインデックス（Notion由来・Excel由来問わず）を **Excel + 個別.mdファイルのZIP** として書き出せる（`export_pageindex_as_excel_bundle()`）。出力Excelの書式は `input: "excel"` のインポート互換。

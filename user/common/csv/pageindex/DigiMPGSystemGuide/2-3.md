# RAGフィルタリングとメタデータ検索

## 概要
ChromaDBへのクエリ時に、FILTER条件やMETA_SEARCHによるメタデータベースの絞り込み・ボーナス補正を行う機能。Private Modeによるプライベートデータの除外もここで制御される。

## 仕組み

### FILTERの構築（`_build_where_limitation`）
エージェントJSONのKNOWLEDGE内各DATAに定義されたFILTER設定から、ChromaDB用のwhere条件を組み立てる。

- **FILTER.PATTERN**: 複数条件の結合方法（`"and"` / `"or"`）
- **CONDITION の種類**:
  - `SERVICE_INFO`: `exec_info`のSERVICE_IDでフィルタ。ITEMSで対象メタデータ列を指定。
  - `USER_INFO`: `exec_info`のUSER_IDでフィルタ。
  - `DEFINE_CODE`: エージェントの`DEFINE_CODE`辞書のキーとChromaDBメタデータ列のマッピング。
    - 各値は **文字列または配列** が許容される。文字列なら`$eq`、配列なら`$in`でwhere句を生成。
    - 空配列はそのコードのフィルタを **無効化**（条件から除外）。
- 各CONDITION内部にもPATTERNを指定可能（複数ITEMSの結合方法）。

### META_SEARCH（`_query_collection_single`）
日付条件付きの追加クエリを実行し、結果の類似度スコアに`BONUS`係数を乗算して優遇する。`meta_searches`リストからDATE範囲を取得し、`create_date_ts`（UNIXタイムスタンプ）に対して`$gte`/$lte`条件を構築する。通常クエリの結果と統合される。

### Private Mode
`_process_rag_data()`で`private_mode=True`の場合、where条件に`{"private": {"$ne": True}}`を追加し、privateフラグが付いたチャンクを除外する。

### TEXT_LIMITS
`select_rag_vector()`でチャンク選択時、累計文字数が`TEXT_LIMITS`を超えないよう制御する。

## 設定場所
- FILTER定義: エージェントJSON の `KNOWLEDGE[].DATA[].FILTER`
- META_SEARCH: エージェントJSON の `KNOWLEDGE[].DATA[].META_SEARCH`（CONDITION, BONUS）
- TEXT_LIMITS: エージェントJSON の `KNOWLEDGE[].TEXT_LIMITS`
- Private Mode: 呼び出し元から`private_mode`パラメータで制御

## 設定例
```json
"FILTER": {
  "PATTERN": "and",
  "CONDITION": {
    "USER_INFO": {"PATTERN": "and", "ITEMS": ["user_id"]},
    "DEFINE_CODE": {"PATTERN": "or", "CODES": {
      "EMP_CODE": "emp_code",
      "DEPT_CODE": "dept_code"
    }}
  }
},
"META_SEARCH": {"CONDITION": ["DATE"], "BONUS": 0.5}
```

エージェント側 `DEFINE_CODE`（リスト対応）:
```json
"DEFINE_CODE": {
  "COMPANY_CODE": [],
  "DEPT_CODE": ["D001", "D002"],
  "EMP_CODE": ["emp001"]
}
```
→ where: `{"$or": [{"emp_code": {"$in": ["emp001"]}}, {"dept_code": {"$in": ["D001","D002"]}}]}`（空配列のCOMPANY_CODEは自動的に除外）

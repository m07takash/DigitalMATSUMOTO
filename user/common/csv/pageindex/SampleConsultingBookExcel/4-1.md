# ビジネスモデルキャンバス（BMC）

事業の構造を9つのブロックに分解して1枚で可視化するフレームワーク（Osterwalder）。仮説立案・既存事業の整理・新規事業のコミュニケーションに広く使われる。

## 9つのブロック

| 領域 | ブロック | 問い |
|------|---------|------|
| 価値 | **Value Propositions**（価値提案） | どんな価値を、誰に提供するか |
| 顧客側 | **Customer Segments**（顧客セグメント） | 誰のための価値か |
| 顧客側 | **Channels**（チャネル） | どう届けるか |
| 顧客側 | **Customer Relationships**（顧客関係） | どう関係を築くか |
| 収益 | **Revenue Streams**（収益の流れ） | 何で稼ぐか |
| 自社側 | **Key Resources**（主要リソース） | 何が必要か |
| 自社側 | **Key Activities**（主要活動） | 何をするか |
| 自社側 | **Key Partners**（パートナー） | 誰と組むか |
| コスト | **Cost Structure**（コスト構造） | 何にお金がかかるか |

## 描く順序の推奨

1. **Customer Segments**（誰のために？）
2. **Value Propositions**（何の価値を？）
3. **Channels** + **Customer Relationships**（どう届け、どう関係を結ぶ？）
4. **Revenue Streams**（どう稼ぐ？）
5. **Key Resources** + **Key Activities** + **Key Partners**（どう作る？）
6. **Cost Structure**（いくらかかる？）

「右半分（顧客・価値）→ 左半分（オペレーション）→ 下（収益とコスト）」で描くと一貫する。

## 整合性のチェック観点

- 価値提案は **顧客セグメントの真のJob** に答えているか
- チャネルとCustomer Relationshipsの粒度が合っているか
- 収益とコストのバランスが現実的か（黒字構造になっているか）
- Key Activities / Resources は本当に **差別化の源泉** か

## バリュープロポジションキャンバスとの併用

BMCの「Value Propositions」と「Customer Segments」だけを拡大して詳細化したのが Value Proposition Canvas。Job／Pain／Gain（顧客側）と Products・Services／Pain Relievers／Gain Creators（自社側）の整合を見る。

## よくある失敗

- 9マスを **埋めることが目的化** する。仮説検証の起点であることを忘れない
- 顧客セグメントが抽象的（「中小企業」「30代女性」）で、価値提案がぼやける
- 既存事業の **整理** に使うと現状肯定で終わる。新規事業では「**変えるべきブロック**」を明示する

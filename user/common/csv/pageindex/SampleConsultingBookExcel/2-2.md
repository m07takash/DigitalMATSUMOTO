# ロジックツリー

問題を「MECE（漏れなくダブりなく）」に分解し、原因・打ち手・KPIを構造化するための基本ツール。

## 主な3タイプ

| タイプ | 用途 | 例 |
|--------|------|-----|
| **Whyツリー** | 原因深掘り | 「なぜ売上が落ちたか」→ 客数 / 客単価 → さらに分解 |
| **Howツリー** | 打ち手展開 | 「どうやって売上を上げるか」→ 新規獲得 / 既存維持 / アップセル |
| **Whatツリー** | 構造把握 | 「売上とは何か」→ 客数 × 客単価 |

## 設計のコツ

1. **頂点の問いを名詞句ではなく文で書く**：「コスト削減」ではなく「製造原価をいくら下げるか」
2. **同階層は同じ切り口で揃える**：客層別 / 製品別を混在させない
3. **3段以上は深追いしない**：分解しすぎると粒度が荒くなり打ち手が出ない
4. **ボトムまで降りたらKPI化**：定量化できないノードは打ち手にならない

## よくある失敗

- MECEに見えて実は重複している（例：「新規顧客」と「若年層」）
- 抽象度がバラバラ（戦略レベルと実行レベルが同階層）
- 出てきた要素を全部追おうとして優先度をつけない

# カスタマージャーニーマップ（CJM）

顧客が商品・サービスを認知し、購入・利用・継続するまでの体験を時系列で可視化するフレームワーク。

## 構成要素

| 要素 | 内容 |
|------|------|
| **フェーズ** | 認知 → 検討 → 購入 → 利用 → 継続/推奨 |
| **行動** | 各フェーズで顧客が取る行動 |
| **タッチポイント** | 顧客が企業と接する場所（広告、Web、店舗、サポート…） |
| **思考・感情** | 顧客の頭の中・気持ちの動き |
| **ペインポイント** | 不満・障害が発生する箇所 |
| **機会** | 体験を改善できる打ち手 |

## ペルソナと組み合わせる

CJMはペルソナ（具体的な顧客像）とセットで作る。「誰の」ジャーニーかを定めないと、平均化された無味無臭のマップになりやすい。

## 作り方の手順

1. **対象ペルソナを明確化**（属性・課題・行動原理）
2. **フェーズの定義**（業界・商材により異なる。BtoBは検討期間が長く、利用後フェーズが重要）
3. **タッチポイントの棚卸し**（既存接点を全部書き出す）
4. **顧客視点で行動と感情を埋める**（社内視点で書かない。インタビュー・観察で検証）
5. **ペイン・機会を特定**（感情曲線の谷を狙う）
6. **施策を検討**（タッチポイント単位ではなく、ジャーニー全体の体験設計）

## 感情曲線

各フェーズの感情をプラス／マイナスで連続的にプロット。「期待が高まったあとに裏切られる」局面が最大のペインになりやすい。

## サービスブループリントとの違い

CJMが「顧客から見える表側」を描くのに対し、**サービスブループリント** は「企業側のオペレーション（裏側）」まで含めて描く。両者をセットにすると、顧客接点の体験と業務オペレーションの双方を整合させられる。

## よくある失敗

- 自社視点で書く（CRM上の顧客状態 ≠ 顧客の実体験）
- タッチポイントの羅列で終わり、感情・思考が浅い
- ペルソナを設定せず「平均的顧客」を描いてしまう
- 一度作って終わり。**顧客行動が変われば毎年見直す** 必要がある

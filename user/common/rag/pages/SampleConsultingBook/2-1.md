# 業務プロセスの可視化と分析

## 概要

業務改革の第一歩は、現行プロセスを正確に可視化し、定量的に分析することである。「見えないものは改善できない」という原則に基づき、業務の流れ・担当・所要時間・コストを明らかにする手法を解説する。

## BPMN記法による業務フロー作成

BPMN（Business Process Model and Notation）は、業務プロセスを標準的な記法で表現するための国際規格である。

**主要な記号：**
- **イベント（丸）：** プロセスの開始・中間・終了を表す
- **アクティビティ（角丸四角）：** 実行される作業・タスクを表す
- **ゲートウェイ（ひし形）：** 分岐・合流の判断ポイントを表す
- **フロー（矢印）：** アクティビティ間の順序関係を表す

**作成手順：**
1. 対象業務の範囲を定義する（スコープ設定）
2. 主要なアクティビティを洗い出す
3. 時系列順に配置し、フローで接続する
4. 分岐条件・例外処理を追加する
5. 関係者にレビューを依頼し、正確性を検証する

## スイムレーンの活用

スイムレーンは、業務フロー図を担当者・部門別に区分けする手法である。

**効果：**
- 部門間の受け渡しポイント（ハンドオフ）が一目で分かる
- 責任の所在が明確になる
- 部門間の重複作業や手戻りが発見しやすくなる

**作成のコツ：**
- 横軸にプロセスの流れ、縦軸に担当部門を配置する
- ハンドオフが多い箇所は改善候補として注目する
- システム処理もひとつのレーンとして記述する

## ボトルネック特定の方法

業務プロセス上のボトルネックを特定するための4つの視点を紹介する。

1. **待ち時間の長い工程：** 承認待ち、確認待ちで滞留が発生していないか
2. **手戻りの多い工程：** 差し戻しや再作業が頻発していないか
3. **属人化した工程：** 特定の個人に依存し、その人が不在だと止まる工程はないか
4. **手作業の多い工程：** エクセルへの転記やダブルチェックなど非効率な手作業がないか

## 定量分析の方法

### タイムスタディ
各アクティビティの所要時間を計測し、プロセス全体のリードタイムとタッチタイム（実作業時間）を算出する。タッチタイム比率（タッチタイム / リードタイム）が低い場合、待ち時間の削減余地が大きい。

### コスト分析
各工程の人件費・システム費・外注費を集計し、コスト構造を可視化する。ABC（Activity Based Costing）の考え方を応用すると精度が上がる。

### 頻度・量分析
月間処理件数、ピーク時の処理量、季節変動などを把握し、リソース配分の最適化に活用する。

## まとめ

業務プロセスの可視化と分析は、改革施策の方向性を決定づける重要な工程である。時間をかけてでも正確に現状を把握することが、効果的な改革の基盤となる。
